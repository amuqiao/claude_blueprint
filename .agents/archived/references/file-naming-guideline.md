# File Naming Guideline

## 文档职责

- 本文档用于承载文件命名、后缀与重命名约束的真源定义。
- 它回答的是“文件该怎么命名、哪些临时命名应避免、什么时候不应批量重命名”。

## 适用范围

- 新建文件、重命名文件、统一同主题文档后缀时读取。
- 需要判断是否应沿用已有命名惯例时读取。
- 不处理文档表达、任务推进、测试验证或安全边界。

## 核心规则

- 文件名应优先保持稳定、可预测、便于链接、引用与迁移。
- 默认不要在文件名中使用空格。
- 新建文档默认优先使用英文小写加连字符，例如：`api-overview.md`、`deployment-checklist.md`。
- 中文文件名可以保留，但词间分隔优先使用 `-`。
- 避免使用 `copy`、`final`、`最新版`、`new` 这类临时词。
- 同主题的 Prompt 文件统一使用 `-Prompt.md` 后缀。
- 同主题的 Checklist 文件统一使用 `-Checklist.md` 后缀。
- 若仓库内已有明确命名惯例，应优先沿用，保持同层级命名一致。
- 除非用户明确要求，否则不要批量重命名已有文件。

## 与运行时规则的关系

- `rules/file-naming.md` 只保留默认加载的短规则。
- 如果未来需要补充跨目录命名策略、迁移规则或例外判断，应先更新本文件，再回写摘要。

## 缺失或冲突时怎么处理

- 如果 `rules/file-naming.md` 与本文件冲突，以本文件为准。
- 如果某条命名约束只适用于特定目录或特定类型文档，应在对应专项真源中说明，不直接污染通用命名规则。
