# Security Guideline

## 文档职责

- 本文档用于承载通用安全边界与敏感信息处理规则的真源定义。
- 它回答的是“哪些内容不能碰、哪些输入必须校验、哪些凭据不能暴露”。

## 适用范围

- 需要判断是否涉及敏感目录、敏感凭据或输入安全时读取。
- 需要决定示例代码、日志、文档中能否出现真实敏感信息时读取。
- 不处理任务推进、测试口径、Git 提交或文档表达结构。

## 核心规则

- 禁止硬编码密钥、token、密码或其他敏感凭据；优先从环境变量或安全配置读取。
- 涉及用户输入时，必须做边界校验；涉及数据库查询时，必须使用参数化方式。
- 未经用户明确要求，不读取、不修改敏感目录或敏感凭据文件，如 `~/.ssh`、`~/.aws`、shell rc、密钥文件等。
- 不在日志、示例代码、文档中泄露真实密钥、token、cookie 或内部地址。

## 与运行时规则的关系

- `rules/security.md` 只保留最小硬边界摘要。
- 如果未来需要更具体的凭据治理、目录访问分级或发布安全约束，应先在本文件扩展，再回写摘要。

## 缺失或冲突时怎么处理

- 如果运行时摘要与本文件冲突，以本文件为准。
- 如果某条内容已经属于仓库特定流程或发布策略，应迁到更具体的真源文档，而不是继续堆在本文件。
