# Testing Guideline

## 文档职责

- 本文档用于承载测试、验证与完成声明规则的真源定义。
- 它回答的是“什么情况下才能说已完成、已修复、测试通过，以及最小验证应如何成立”。

## 适用范围

- 需要决定最小验证范围、完成声明口径或测试说明方式时读取。
- 需要判断是否可以接受 mock、是否必须提供运行输出时读取。
- 不处理任务推进、安全边界、Git 提交或命名规则。

## 核心规则

- 声明“已完成”“已修复”或“测试通过”前，必须提供本次会话内的新鲜验证依据。
- 声明“测试通过”前，必须展示实际运行输出，或明确说明本次采用的验证方式。
- 有相关测试时，优先运行与改动直接相关的测试；若仓库明确要求全量测试，提交前遵守该要求。
- 无法运行测试时，必须明确说明原因，不得假设通过。
- 单元测试中涉及外部服务时，优先使用 mock 或替身，避免真实依赖。

## 与运行时规则的关系

- `rules/testing.md` 只保留运行时高频摘要。
- 如果未来需要扩展验证类型、示例模板或完成声明样例，应先在本文件扩展，再回写摘要。

## 缺失或冲突时怎么处理

- 如果运行时摘要与本文件冲突，以本文件为准。
- 如果某项要求实际上属于工作流或安全边界，应迁回对应真源。
