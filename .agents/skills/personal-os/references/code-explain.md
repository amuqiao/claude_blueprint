# Code Explain

## 文档定位

代码讲解文档是**可选产出**，仅在明确有讲解需求时创建。

选题原则：

- 只讲难点
- 只讲混淆点
- 只讲关键设计

通常不需要讲：

- 命名已经能自解释的逻辑
- 目标读者已知的通用模式
- 单纯 CRUD、胶水代码、没有设计张力的样板代码

推荐命名：

- `{主题}代码讲解.md`

## 默认结构

1. 心智模型
2. 关键逻辑单元讲解
3. 执行顺序（按需）

### 心智模型

第一节固定为“心智模型”。

目的：

- 帮读者建立理解后续内容所需的认知前提
- 先说明这段代码到底在解决什么问题
- 降低后续细节理解门槛

内容可以是：

- 运行上下文
- 数据结构理解
- 领域概念
- 状态流或输入输出关系

### 关键逻辑单元讲解

每个讲解单元按以下顺序判断：

1. 问题段
   原因不显而易见时，再写它解决什么问题。
2. 原理段
   至少包含一种更适合理解的表达方式：
   - 公式
   - 示意图思路
   - 伪代码
   - 状态变化描述
3. 代码段
   - 只贴关键 3 到 10 行
   - 不贴完整函数
   - 重点解释 WHY，不是 WHAT
4. 参数段
   有参数时说明含义和如何影响行为。

### 执行顺序

满足以下任一条件时补充：

- 多个单元有固定先后顺序
- 前一单元的输出是后一单元的输入
- 顺序不能调换

## 输出原则

- 先讲整体理解，再讲细节
- 优先解释 WHY
- 只保留真正值得讲的单元
- 不要堆满代码片段
- 不要把讲解文档写成逐行注释版源码
