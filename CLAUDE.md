# 全局开发规范

> **位置**：~/.claude/CLAUDE.md（系统级，应用于所有项目）
> 
> 本文件定义跨项目通用的 AI 行为约束。
> 各项目在自己根目录的 CLAUDE.md 中定义专属内容（路径映射、技术栈细节、项目特定规则）。
> 
> **组织方式**：
> - 核心约束：本文件
> - 模块化规则：rules/ 目录（当前仅保留最小示例，用于验证 rules 机制可用；不是默认大规模拆分层）
> - 细节规范：skills/ 目录（按任务类型加载）

---

## 通用分层架构约束

这套分层原则适用于所有全栈项目，具体目录名在项目 CLAUDE.md 中映射。

### 后端四层

| 层 | 职责 | 禁止 |
|----|------|------|
| 入口层 | 接收外部触发（HTTP/队列），校验参数，委托应用层 | 写业务逻辑；直接操作数据库 |
| 应用层 | 业务流程编排，调用数据访问层和适配器 | 直接操作数据库；感知传输协议细节 |
| 数据访问层 | 封装所有数据库读写，返回数据契约对象（非 ORM 对象） | 包含业务判断；调用应用层 |
| 基础设施层 | 外部服务连接、配置、工具适配 | 包含业务逻辑 |

层间调用方向：入口层 → 应用层 → 数据访问层 → 基础设施层（单向，不可逆）。

### 前端五层

| 层 | 职责 | 禁止 |
|----|------|------|
| 页面层 | 路由入口，组合组件和状态 | 直接调用 API；写业务逻辑 |
| 组件层 | 可复用 UI，接受 props | 直接调用 API；持有全局状态 |
| 状态层 | 全局状态管理，调用服务层 | 直接发起 HTTP 请求 |
| 服务层 | 封装所有 API 调用 | 持有状态；渲染 UI |
| 基础设施层 | HTTP 客户端、实时连接工具 | 包含业务逻辑 |

## 通用工作流约束

- **开发新功能前**：先有设计文档（至少覆盖问题定义、目标、数据模型），再动代码
- **代码落库后**：立即更新 `docs/design/INDEX.md` 中该模块的同步状态
- **UI 类任务**：先画极简 ASCII 布局理解结构 → 再建 HTML mock → 再写/更新设计文档
- **有前端的 UI PR 改动用户路径或按钮语义**：必须同步更新 mock.html
- **Smoke test**：新功能先跑核心流程确认无问题，再交付/发版
- **迁移草案**：不产出独立文件，schema 决策直接写进设计文档的数据模型节

## 写作规范路由

**所有文档输出任务**必须遵守 `rules/writing.md` 的约束：
- 整体理解优先的表达底线
- 结构化表达与图示使用边界

**新建正式文档、重构文档可读性、或 Review 文档结构**时，调用 `~/.claude/skills/writing/SKILL.md`：
- 落实固定输出契约（`# 一级标题 + 4 行引用块`）
- 先建立整体理解，再组织主干结构
- 处理层级关系、边界区别和结构化表达
- 判断何时使用清单、表格、文本树或 Mermaid
