⚠ 重要说明 — 文档未明确或常见误区

  • hooks 的正确配置方式：【官方标准】 hooks 不是一个独立目录，而是写在 settings.json 的 "hooks" 字段中。 shell 脚本文件可放任意位置，在 settings.json 中用路径引用。 支持事件：PreToolUse / PostToolUse / Notification / Stop / PreCompact / InstructionsLoaded 等。
  • 项目级无独立 plugins/ 目录：【官方标准】 plugins 仅在系统级 ~/.claude/plugins/ 存在；项目通过 settings.json 的 plugins 字段配置，指定作用范围（user 或 project）。
  • commands/ 与 skills/ 关系：【官方标准】 二者现已合并为同一机制。commands/*.md 是单文件形式的 skill；skills/<name>/SKILL.md 支持打包多个支持文件。现有 commands/ 文件无需迁移，继续有效。
  • statusline： 官方文档有 statusline 功能页面，但其配置方式是通过 settings.json 字段控制，不是一个独立的配置文件（如某些示例中出现的 statusline 文件是非官方写法）。
  • Auto memory 路径：【官方标准】 固定存储在 ~/.claude/projects/<hash>/memory/；autoMemoryDirectory 字段可自定义路径（仅接受 user 级设置，不接受 project/local 设置，防止 repo 重定向写入敏感路径）。需 v2.1.59+。
  • CLAUDE.md 在子目录中的加载时机：【官方标准】 工作目录上级的 CLAUDE.md 在启动时全量加载；子目录中的 CLAUDE.md 在 Claude 读取该子目录文件时按需加载。
