 # AI 漫剧字幕与配音替换扫盲及方案文档计划

  ## Summary

  你的需求是：已有一段外语 srt 和一段带原字幕、原配音的 mp4，希望以 srt 为基本处理单元，把对白翻译/改写成目标语言，同时替换画面字幕和配音，并尽量保持原角色音色相
  似。核心难点不是单纯“翻译字幕”，而是字幕分段、语种长度差异、配音时长、角色情绪、音色相似度、原视频硬字幕处理和人工质检之间的协同。

  建议拆成 3 篇文档，因为这个主题同时包含认知扫盲、技术坐标和工程方案。集中成一篇会过长，也不利于后续把方案演进为平台能力。

  ## 三篇文档结构

  1. 《AI 漫剧字幕与配音替换基础扫盲》
      - 用“主题基础扫盲方法论”建立对象：srt、对白、字幕、配音、音色克隆、时间轴、硬字幕/软字幕、音轨混音分别是什么。
      - 解释为什么不能把问题简化为“翻译 SRT + TTS”：不同语言长度不同，配音必须服从时间窗口，字幕也要服从阅读速度和画面节奏。
      - 明确边界：本需求不是纯 ASR、不是纯 TTS、不是纯视频剪辑，而是一个多模态后期生产流水线。
  2. 《AI 配音替换技术坐标与方案选型》
      - 用“主题认知坐标建模方法论”比较三类路线：第三方 API、开源自部署、自研训练。
      - 第三方方向以 ElevenLabs 这类 dubbing/API 产品作为 MVP 参照；它官方支持音视频
        dubbing、说话人保留、字幕/翻译编辑和局部重生成能力，适合快速验证流程，但有成本和供应商依赖。(elevenlabs.io
        (https://elevenlabs.io/docs/capabilities/dubbing))
      - 开源方向按能力拆解：WhisperX 可用于 ASR、词级时间戳和说话人分离；OpenVoice、CosyVoice、GPT-SoVITS 可作为音色克隆/TTS 候选。(github.com
        (https://github.com/m-bain/whisperX))
      - 自研方向只建议作为中后期路线：先沉淀数据、标注、质检指标和用户工作流，再考虑训练或微调专用模型。
  3. 《AI 漫剧平台 MVP 落地方案》
      - 推荐路线：混合路线 + 人审半自动工作流。
      - 后端用你熟悉的 FastAPI 做任务编排：上传 mp4/srt、解析字幕、生成翻译稿、生成候选配音、对齐时长、混音、烧录/外挂字幕、人工审核、导出成片。
      - MVP 先调用第三方或成熟开源组件验证端到端链路，同时把 ASR、翻译、TTS、音频对齐、视频渲染都抽象成可替换模块。
      - 人工工作台是必要能力：允许编辑字幕文本、合并/拆分 SRT 段落、调整说话速度、重生成单句配音、替换角色音色、预览局部片段。

  ## 推荐方案

  优先做 “SRT 驱动的半自动配音替换流水线”，不要一开始追求全自动 end-to-end dubbing。

  mp4 + source.srt
  -> 翻译/本地化改写
  -> 分段合并或拆分
  -> 替换字幕
  -> 人工质检
  -> 导出目标语言版本

  技术选型默认建议：

  - MVP：优先接第三方 TTS/dubbing 能力，快速验证质量、成本和用户流程。
  - 开源试验线：用 WhisperX 做辅助对齐/说话人识别；用 OpenVoice、CosyVoice、GPT-SoVITS 做音色相似度和多语言 TTS 对比实验。OpenVoice
    明确强调跨语言音色克隆和声音风格控制；CosyVoice 提供多语言语音生成、训练、推理和部署能力；GPT-SoVITS 支持 5 秒 zero-shot、1 分钟 few-shot
    以及中英日韩粤等跨语言推理。(github.com (https://github.com/myshell-ai/OpenVoice))
  ## Test Plan

  - 用 3-5 条短漫剧片段测试：英语、日语、韩语、法语、印尼语各至少一条。
  - 每条片段覆盖：单人对白、多人对话、大段对白、快速对白、带背景音乐/音效片段。
  - 验收指标：
      - 字幕是否完整替换。
      - 配音是否落在原 SRT 时间窗附近。
      - 是否支持单句重生成。
      - 角色音色是否稳定。
      - 原背景音乐/音效是否可保留。
      - 人工是否能修正分段和翻译。

  ## Assumptions

  - 文档面向你自己和后续开发/选型协作者，默认中文写作。
  - 第一阶段目标是平台 MVP 和技术可行性验证，不是直接训练一个完整自研 TTS/配音大模型。
  - 默认选择“三篇文档 + 混合路线 + 人审半自动工作流”。
  - 若原视频字幕是硬字幕，第一版优先用遮罩/字幕区域覆盖或重新排版字幕；视频修复/inpaint 可作为后续增强能力。
