## 1. 多人重叠对话

### 场景描述

两个或多个角色同时说话，音轨有重叠，字幕时间戳也可能重叠。

### 问题

- 人声分离无法完全区分重叠的人声
- TTS 单独生成每个角色后，混音时会叠加，但缺少原始的空间感和音量关系
- 字幕渲染时可能出现重叠显示

### 处理策略

| 阶段 | 策略 | 说明 |
| --- | --- | --- |
| **MVP** | 标记为"重叠段"，人工处理 | 自动检测 srt 时间重叠，标记 `overlap: true`，提示人工决策 |
| **V1** | 音量衰减叠加 | 生成的配音按原始音量比例混音，模拟空间感 |
| **V2** | 双轨渲染 | 支持多条 segment 在同一时间段独立混音，保留左右声道差异 |
| **V3** | 智能分离 + 重组 | 使用更高级的人声分离模型（如 Spleeter 5-stem），分离后按角色重组 |

### MVP 实现

```python
# 检测重叠
def detect_overlap(segments):
    overlaps = []
    for i, seg in enumerate(segments[:-1]):
        next_seg = segments[i + 1]
        if seg.end_time_ms > next_seg.start_time_ms:
            overlaps.append({
                "seg1": seg.id,
                "seg2": next_seg.id,
                "overlap_ms": seg.end_time_ms - next_seg.start_time_ms
            })
            seg.overlap = True
            next_seg.overlap = True
    return overlaps

# 人工处理入口
# 1. 调整时间窗，错开 segment
# 2. 手动指定音量比例
# 3. 接受重叠，混音时叠加
```

## 2. 静音段处理

### 场景描述

原视频中有较长的静音段（如角色思考、镜头切换、纯背景音乐）。

### 问题

- 如果 srt 没有覆盖静音段，生成的音轨会缺失这些时间
- 如果强行填充静默，可能破坏节奏感

### 处理策略

| 方案 | 说明 | 适用场景 |
| --- | --- | --- |
| **保留原音轨静音段** | 检测 srt 空白时间段，保留原音轨该段不替换 | 静音段有背景音乐或音效 |
| **填充静默** | 生成音轨中该段填充空白音频 | 纯静音，无背景音 |
| **标记提示** | 标记静音段供人工确认 | 不确定时，交给人工决策 |

### 实现

```python
def detect_silence_gaps(segments, video_duration_ms):
    gaps = []
    prev_end = 0
    for seg in sorted(segments, key=lambda s: s.start_time_ms):
        if seg.start_time_ms - prev_end > 1000:  # >1s 视为静音段
            gaps.append({
                "start_ms": prev_end,
                "end_ms": seg.start_time_ms,
                "duration_ms": seg.start_time_ms - prev_end
            })
        prev_end = seg.end_time_ms

    # 检查最后一段
    if video_duration_ms - prev_end > 1000:
        gaps.append({
            "start_ms": prev_end,
            "end_ms": video_duration_ms,
            "duration_ms": video_duration_ms - prev_end
        })

    return gaps

# 混音时对静音段的处理
def mix_audio_with_gaps(segments, gaps, original_audio):
    for gap in gaps:
        # 从原音轨提取该段
        gap_audio = extract_audio_segment(original_audio, gap["start_ms"], gap["end_ms"])
        # 保留到目标音轨
        insert_audio_segment(gap_audio, gap["start_ms"])
```

## 3. 背景音过大

### 场景描述

原视频背景音乐或音效音量过大，压过对白，导致：
- 人声分离质量下降
- 配音和背景音音量不平衡

### 处理策略

| 方案 | 说明 | 工具 |
| --- | --- | --- |
| **人声分离 + 动态混音** | 分离人声和背景音后，对背景音做动态压缩（ducking），对白时段自动降低背景音 | Demucs + pydub ducking |
| **背景音降噪** | 对背景音轨做降噪处理 | `noisereduce` 库 |
| **手动调整混音比例** | 允许用户调整对白和背景音的音量比例 | 滑块控制 -12dB 到 +6dB |

### 动态压缩（Ducking）实现

```python
from pydub import AudioSegment

def apply_ducking(dialogue_audio, background_audio, reduction_db=-12):
    """
    对白时段自动降低背景音
    """
    ducked_bg = background_audio.copy()

    for seg in segments:
        start_ms = seg.start_time_ms
        end_ms = seg.end_time_ms

        # 对白时段降低背景音
        ducked_bg = (
            ducked_bg[:start_ms] +
            ducked_bg[start_ms:end_ms] + reduction_db +  # 降低 12dB
            ducked_bg[end_ms:]
        )

    # 叠加对白和降低后的背景音
    mixed = ducked_bg.overlay(dialogue_audio)
    return mixed
```

## 4. 特殊音效处理

### 场景描述

角色声音有特殊音效，如：
- 机器人音（电子音）
- 回声（echo）
- 电话滤波效果
- 变声效果

### 问题

TTS 生成的是干净人声，无法直接复现原音轨的音效。

### 处理策略

| 阶段 | 策略 | 说明 |
| --- | --- | --- |
| **MVP** | 标记为特殊音效，人工后期处理 | 检测到异常音色时标记，导出后人工加音效 |
| **V1** | 音效库应用 | 预设常见音效（robot、echo、phone），生成后自动应用 |
| **V2** | 音效提取 + 迁移 | 从原音轨提取音效特征，迁移到生成的配音上 |

### V1 音效库示例

```python
# 使用 pydub 和 ffmpeg 应用音效
def apply_robot_effect(audio_segment):
    # 机器人音：pitch shift + chorus
    return audio_segment.pitch_shift(semitones=-3).chorus()

def apply_echo_effect(audio_segment, delay_ms=200, decay=0.5):
    # 回声：延迟叠加
    echo = audio_segment - 10  # 降低音量
    return audio_segment.overlay(echo, position=delay_ms, gain_during_overlay=decay)

def apply_phone_effect(audio_segment):
    # 电话音：带通滤波 300Hz - 3400Hz
    return audio_segment.low_pass_filter(3400).high_pass_filter(300)
```

### Segment 数据模型扩展

```python
class Segment(Base):
    # ... 其他字段
    audio_effect = Column(String, nullable=True)  # "robot" / "echo" / "phone" / null
    effect_parameters = Column(JSON, nullable=True)  # {"delay_ms": 200, "decay": 0.5}
```

## 5. 长视频处理

### 场景描述

视频时长超过 30 分钟，segment 数量超过 500 条。

### 问题

- 批量任务耗时过长（可能数小时）
- 内存占用大
- 中途失败后难以恢复

### 处理策略

| 方案 | 说明 |
| --- | --- |
| **分段处理** | 按时间或 segment 数量分段，每段独立处理后合并 |
| **增量处理** | 只处理修改过的 segment，未修改的复用缓存 |
| **断点续传** | 任务失败后从上次中断点继续，不重新开始 |
| **并行处理** | 多个 worker 并行处理不同 segment |

### 实现：分段处理

```python
def split_project_by_time(project, segment_duration_minutes=10):
    """
    按时间分段，每段 10 分钟
    """
    segments = project.segments
    duration_ms = segment_duration_minutes * 60 * 1000

    chunks = []
    current_chunk = []
    chunk_start = 0

    for seg in sorted(segments, key=lambda s: s.start_time_ms):
        if seg.start_time_ms - chunk_start > duration_ms:
            chunks.append(current_chunk)
            current_chunk = [seg]
            chunk_start = seg.start_time_ms
        else:
            current_chunk.append(seg)

    if current_chunk:
        chunks.append(current_chunk)

    return chunks

# 每个 chunk 独立处理
for i, chunk in enumerate(chunks):
    process_chunk(project_id, chunk, chunk_index=i)
```

### 断点续传

```python
class Job(Base):
    # ... 其他字段
    progress = Column(Float, default=0.0)  # 0.0 - 1.0
    checkpoint = Column(JSON, nullable=True)  # 记录已完成的 segment_ids

def resume_job(job_id):
    job = get_job(job_id)
    completed_segment_ids = job.checkpoint.get("completed", [])

    # 只处理未完成的 segment
    remaining_segments = [
        seg for seg in job.project.segments
        if seg.id not in completed_segment_ids
    ]

    for seg in remaining_segments:
        process_segment(seg)
        # 更新 checkpoint
        completed_segment_ids.append(seg.id)
        job.checkpoint["completed"] = completed_segment_ids
        job.save()
```

## 6. 极短对白段

### 场景描述

单条 segment 时长 <0.5s，如短促的"啊"、"嗯"、"哈"。

### 问题

- TTS 生成时可能无法压缩到如此短的时长
- 即使生成，音质可能失真

### 处理策略

| 方案 | 说明 | 适用场景 |
| --- | --- | --- |
| **保留原音** | 极短段直接保留原音轨该段 | 只是语气词，不影响理解 |
| **合并相邻段** | 与前后段合并为一个更长的 segment | 语义上连续 |
| **语速拉伸** | TTS 生成后极限压缩（time-stretch up to 1.5x） | 必须替换且可压缩 |
| **静默填充** | 如果文本为空或标点，直接填充静默 | 纯停顿 |

### 检测与标记

```python
def detect_extreme_short_segments(segments, threshold_ms=500):
    short_segs = []
    for seg in segments:
        if seg.duration_ms < threshold_ms:
            seg.duration_status = "extreme_short"
            short_segs.append(seg)
    return short_segs
```

## 7. 专有名词错读

### 场景描述

角色名、地名、招式名等专有名词被 TTS 错读，如"鸣人"被读成"ming ren"而非"naruto"。

### 处理策略

| 方案 | 说明 | 支持度 |
| --- | --- | --- |
| **术语表替换** | 翻译后用固定译名替换，如"鸣人" → "Naruto" | 所有 provider |
| **音素标注（SSML）** | 使用 SSML 或 IPA 音标指定读音 | Azure TTS, ElevenLabs 部分支持 |
| **参考音频提供** | 提供正确读音的参考音频片段 | 音色克隆类 provider |
| **人工标注发音词典** | 为项目或角色维护发音词典 | 所有 provider |

### SSML 示例（Azure TTS）

```xml
<speak version="1.0" xmlns="http://www.w3.org/2001/10/synthesis" xml:lang="en-US">
    <voice name="en-US-JennyNeural">
        His name is <phoneme alphabet="ipa" ph="nɑːruːtoʊ">Naruto</phoneme>.
    </voice>
</speak>
```

### 发音词典

```python
class PronunciationDict(Base):
    project_id = Column(String, ForeignKey("projects.id"))
    term = Column(String)  # "鸣人"
    target_language = Column(String)  # "en"
    pronunciation = Column(String)  # "Naruto" 或 IPA "nɑːruːtoʊ"
    provider = Column(String)  # 适用的 provider，null 表示通用
```

## 8. 字幕超长超宽

### 场景描述

目标语言文本过长，超出视频宽度或显示区域。

### 处理策略

| 方案 | 说明 |
| --- | --- |
| **自动换行** | 按字符或单词长度自动换行 |
| **动态字号** | 文本过长时自动缩小字号 |
| **滚动字幕** | 超长文本改为横向滚动显示 |
| **提示缩短** | 标记为 `text_too_long`，提示人工缩短 |

### 实现：动态字号

```python
def calculate_font_size(text, max_width_px, base_font_size=24):
    """
    根据文本长度和视频宽度动态调整字号
    """
    estimated_width = len(text) * base_font_size * 0.6  # 粗略估算
    if estimated_width > max_width_px:
        scale = max_width_px / estimated_width
        return max(int(base_font_size * scale), 16)  # 最小 16px
    return base_font_size
```

## 9. 时间窗不连续

### 场景描述

相邻两条 srt 之间有较大时间间隔（>3s），但语义连续。

### 问题

- 混音时可能出现空白
- 预览时停顿感明显

### 处理策略

| 方案 | 说明 |
| --- | --- |
| **保留间隔** | 间隔内保留原音轨，可能有背景音或音效 |
| **填充静默** | 如果原音轨该段为纯静音，填充空白音频 |
| **提示合并** | 标记为"建议合并"，供人工判断 |

## 10. 软硬字幕混合

### 场景描述

原视频既有软字幕轨，又有烧录的硬字幕。

### 处理策略

| 方案 | 说明 |
| --- | --- |
| **优先软字幕** | 如果软字幕完整，忽略硬字幕 |
| **硬字幕遮罩** | 如果软字幕缺失，检测硬字幕区域并遮罩 |
| **提示人工确认** | 混合场景标记，让用户选择处理方式 |

## 总结

| 场景 | MVP 处理 | V1+ 处理 |
| --- | --- | --- |
| 多人重叠对话 | 标记，人工处理 | 音量衰减叠加，双轨渲染 |
| 静音段 | 保留原音轨静音段 | 自动检测并填充 |
| 背景音过大 | 人声分离 + 手动调整比例 | 动态压缩（ducking） |
| 特殊音效 | 标记，人工后期加音效 | 音效库应用，音效迁移 |
| 长视频 | 分段处理 + 断点续传 | 并行处理 + 增量更新 |
| 极短对白 | 保留原音或合并相邻段 | 语速极限拉伸 |
| 专有名词错读 | 术语表 + 人工标注 | SSML 音素标注 |
| 字幕超长 | 自动换行 + 提示缩短 | 动态字号，滚动字幕 |
| 时间窗不连续 | 保留原音轨间隔 | 智能填充 |
| 软硬字幕混合 | 优先软字幕 | 自动检测并选择策略 |

这些特殊场景在 MVP 阶段优先采用保守策略（标记、提示、人工处理），在后续版本中逐步实现自动化处理。
