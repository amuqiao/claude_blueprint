# 工程开发类方法论

> 本目录存放用于指导项目开发主干、工程推进顺序、模块落地和验证治理的方法论。

## 文档分类

### 通用方法论

- [`技术栈开发手册成型方法论.md`](技术栈开发手册成型方法论.md)
  用于把一种语言、框架或技术栈的开发经验，组织成认知清楚、架构稳定、选型有边界、规范可判断、AI 可执行的长期开发手册。

### 后端工程

- [`后端工程/README.md`](后端工程/README.md)
  后端工程方法论入口，用于区分服务起盘、FastAPI 服务、项目入口、部署发布和新服务能力挂载关系。

### 客户端工程

- [`客户端工程/README.md`](客户端工程/README.md)
  客户端工程方法论入口，用于区分正式方法论、Flutter 0-1 规范、依赖推荐、参考手册和归档资料。

## 维护边界

工程开发类方法论负责回答“项目或模块应该按什么主干推进、服务如何从 0-1 起盘、部署形态如何选择、每个节点产出什么、做到什么程度进入下一步”。

本层 README 只维护一级分类入口和跨目录边界，不维护后端或客户端子目录内部索引。子目录新增、迁移或归档文档时，应优先更新对应子目录的 `README.md`。

如果问题仍停留在需求拆解或方案比较，应回到上层方法论目录选择需求与方案类方法。

如果问题只是具体工具、命令、插件或报错使用说明，应使用理解类中的工具能力使用与排错方法论，不应沉淀到工程开发类。
