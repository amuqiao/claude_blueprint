## 维护边界

客户端工程方法论负责处理客户端应用的版本边界、架构、交互流程、界面表面、状态模型、原型验证、运行验证和回写治理。

它不作为新服务架构龙骨的挂载范式；当项目对象是 Flutter 客户端应用或 Flutter 子工程时，应直接使用本目录下的方法论。

本层目录先通过 README 做轻量分类治理。只有当同类文档继续增长、入口开始拥挤或维护边界再次混乱时，才考虑新增子目录。
