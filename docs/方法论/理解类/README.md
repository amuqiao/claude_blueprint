# 理解类方法论

> 本目录存放用于把主题、对象或工具体系讲清楚的方法论。

## 当前文档

- [`主题基础扫盲方法论.md`](主题基础扫盲方法论.md)
  用于把模糊主题从名称、术语或碎片资料中定住，让它成为可以继续讨论和深入的稳定对象。

- [`主题认知坐标建模方法论.md`](主题认知坐标建模方法论.md)
  用于把已经成形的主题放入关系结构中，解释它的位置、比较对象、机制原因和取舍边界。

- [`工具能力使用与排错方法论.md`](工具能力使用与排错方法论.md)
  用于说明由多个能力、入口、配置和规则组成的工具体系，让读者能区分、会使用、能选择、可排错。

## 维护边界

理解类方法论负责建立对象、结构、关系和使用判断。

理解类内部不是固定执行顺序；应根据对象状态选择基础扫盲、认知坐标建模或工具能力使用与排错，而不是把三者串成流水线。

如果问题已经进入具体需求、候选方案、实现设计或任务拆分，应回到上层方法论目录选择对应方法。
