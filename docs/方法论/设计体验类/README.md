# 设计体验类方法论

> 本目录沉淀 UI、交互、用户心智、信息层级和体验合理性相关的方法论，帮助在不同产品、页面和媒介中判断界面体验是否成立。

## 目录职责

本目录处理的对象不是具体设计稿、具体组件库或单个项目的界面规范，而是可跨项目复用的设计体验判断方法。

适合进入本目录的内容通常具备：

- 能分析 UI 与交互是否合理。
- 能在缺少完整项目文档时自行建立必要心智模型。
- 能处理截图、代码、原型、自然语言描述或真实产品界面。
- 能输出问题判断、调整方向和体验风险，而不绑定某个技术栈。

不适合进入本目录：

- 单个项目的视觉规范。
- 某个组件库的使用说明。
- 一次性页面走查记录。
- 纯工程实现、性能优化或代码重构方法。

## 当前方法论

- [`UI交互合理性扫描方法论.md`](UI交互合理性扫描方法论.md)
  用于像扫描核一样观察页面、组件或流程，自行建立用户心智模型，并判断 UI 和交互在信息层级、状态连续性、操作可理解性和视觉节奏上是否成立。

## 维护边界

新增设计体验方法论时，应先判断它是否处理稳定的体验判断对象，而不是把项目设计经验包装成通用方法。

如果文档主要回答“这个项目应该怎么设计”，应留在项目文档中；如果文档回答“面对任意界面时如何判断体验是否合理”，才适合进入本目录。
