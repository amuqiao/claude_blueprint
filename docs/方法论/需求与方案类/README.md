# 需求与方案类方法论

> 本目录存放用于把需求讲清楚、把方案判断清楚的方法论。

## 当前文档

- [`需求拆解与补充说明方法论.md`](需求拆解与补充说明方法论.md)
  用于把一句话或几段话的粗略需求，拆成可讨论、可确认、可验收的需求说明，但不进入解决方案设计。

- [`方案分析与建议方法论.md`](方案分析与建议方法论.md)
  用于在需求已经拆清楚后，生成候选方案、评估取舍，并给出有前提和风险说明的推荐建议。

## 维护边界

需求与方案类方法论负责从原始诉求走向可确认需求，再从清楚需求走向候选方案和建议判断。

如果问题仍停留在主题、工具体系或对象理解层面，应回到上层方法论目录选择理解类方法。
