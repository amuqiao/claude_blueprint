0
先建心智模型
Claude Code 启动后，"大脑"由四层内容叠加构成
① 系统层
Anthropic 内置指令
每次自动加载，不可见，不可修改
② 规则层
CLAUDE.md + rules/
session 启动时全量加载，始终在后台约束行为
③ 能力层
skills/ · agents/
按需激活：description 匹配 或 显式调用
④ 对话层
你说的话
实时注入，优先级最高，可覆盖规则层
Rules = 约束层（后台持续生效，告诉 Claude 什么不能做）
Skills = 执行层（按需激活，告诉 Claude 怎么完成某类任务）

完整任务流：实现用户登录功能
你说：帮我做用户登录功能 Superpowers 自动触发
Claude 识别到"实现功能"的意图，Superpowers Skill 正文完整加载。
它不直接写代码，而是先引导澄清需求——
Claude 反问： 密码加密方式？JWT 过期时间？失败次数限制？需要"记住我"吗？
Rules 层同时约束（后台，无需你提醒）
workflow.md → 新功能先有设计草案
security.md → 密码必须哈希存储
你说：把规划写到文件里 Planning with Files 显式触发
Skill 正文加载，生成三个持久化文件：
task_plan.md 目标 + 任务分解清单
- [ ] 设计 User 数据模型
- [ ] POST /auth/login 接口
- [ ] useAuth hook
findings.md
技术决策记录
progress.md
当前进度快照
关键：这三个文件是跨 session 的状态存储。上下文压缩或 session 重启后，Claude 读文件即可恢复任务状态，不需要你重新解释背景。
你说：/ralph-loop "按 task_plan.md 逐项完成，完成后输出 <promise>DONE</promise>" Ralph Loop 显式触发
Ralph Loop 接管执行节奏，形成自动循环：
📋
读 task_plan.md
找第一个 [ ] 未完成项
⌨️
实现该任务
写代码
🧪
运行测试 testing.md
展示真实输出，不允许假设性声明
⟳
💾
git commit git.md
feat(auth): add login endpoint
✅
更新 progress.md
勾选 [x]
🪝
Stop Hook 拦截 Ralph Loop
阻止 Claude 退出，重新喂入 prompt
循环直到所有 [x] 完成或输出 <promise>DONE</promise>
上下文接近上限 → /compact 压缩 Planning with Files 恢复价值
对话历史丢失，但文件系统状态完整保留：
📂
你说：读取 task_plan.md 和 progress.md，从中断处继续
Claude 重建任务状态，无缝继续——不需要你重新解释任何背景
功能完成 → 代码整理 Code Simplifier 显式触发
以独立子智能体身份启动，有自己的上下文窗口，专注于：

不改变任何外部行为

💡 suggestion

职责边界一览
CLAUDE.md
最高优先级硬约束，每次 session 无条件生效
自动加载 全局 / 项目 持久：文件
rules/
主题化行为约束，后台持续生效，无需触发
自动加载 支持 paths 持久：文件
skills/
特定任务的操作手册，按需激活，执行完退出
自动匹配 / 显式调用 不持久
agents/
独立子智能体，有自己的上下文窗口和工具权限
主 session 委托 独立上下文
Planning files
跨 session 状态存储，上下文压缩后可读取恢复
Skill 生成 持久：文件系统
hooks
自动化质量门禁，在工具调用前后执行脚本
事件触发 settings.json 配置

最常见的误解
✗ Skills 需要每次手动叫出来才能用
✓ description 写得好的 Skill 会被 Claude 自动匹配触发，Superpowers 这类根本不用你喊名字
✗ Rules 和 Skills 是同一种东西，选一个用就行
✓ Rules 是约束（后台全程生效），Skills 是执行手册（按需激活）。两者不替代，只叠加
✗ 写了 paths 的规则会在需要时才加载进上下文
✓ 所有规则 session 启动时全部扫描进上下文，paths 只是告诉 Claude "这段规则只对特定文件生效"，内容始终在上下文里
✗ 上下文压缩或 session 重启后任务就断了
✓ 用 Planning with Files 把状态写进文件，压缩后读文件恢复，任务无缝继续
✗ CLAUDE.md 越详细越好，写得越多 Claude 越懂你
✓ 超过 100 行后 Claude 开始静默忽略规则。核心约束放 CLAUDE.md，其余拆进 rules/ 分主题管理
