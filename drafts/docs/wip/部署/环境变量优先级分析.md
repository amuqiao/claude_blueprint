⏺ 环境变量优先级分析

  Docker Compose 环境变量优先级（从高到低）
```
  1. docker-compose.yml 中的 environment 部分 ⚠️  最高优先级
     ↓
  2. Shell 环境变量（export LLM_API_KEY=xxx）
     ↓
  3. env_file 文件（.env.dev）
     ↓
  4. Dockerfile 中的 ENV
```

  答案：取决于 .env.dev 的写法

## 核心要点总结

### ✅ 可行的方案（已配置）

```bash
# .env.dev 中的配置
LLM_API_KEY=${LLM_API_KEY:-sk-dev-api-key-here}
```

**工作原理：**
- `${变量名:-默认值}` 是 shell 变量替换语法
- 如果环境变量 `LLM_API_KEY` 存在 → 使用环境变量
- 如果不存在 → 使用默认值 `sk-dev-api-key-here`

### 服务器部署示例

```bash
# 方法 A: 一行命令（推荐临时部署）
LLM_API_KEY=sk-real-xxx ENV_FILE=.env.dev docker-compose up -d --build

# 方法 B: 永久环境变量（推荐生产环境）
export LLM_API_KEY=sk-real-xxx
ENV_FILE=.env.dev docker-compose up -d --build

# 验证
docker-compose exec fastapi printenv LLM_API_KEY
```
