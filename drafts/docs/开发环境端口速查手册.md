# 开发环境端口速查手册

## 查看所有端口占用

查看所有正在监听的端口：
```bash
ss -tlnp
```

查看所有监听端口（含进程名，需 sudo）：
```bash
sudo ss -tlnp
```

用 netstat 查所有监听端口（老系统兼容）：
```bash
netstat -tlnp
```

## 查询指定端口

### 单端口查询

查询指定端口是否被占用：
```bash
ss -tlnp | grep :8000
```

查询指定端口的占用进程详情（含 PID）：
```bash
sudo lsof -i :8000
```

### 批量端口查询

**方法 1：一次扫描 + 多端口匹配**（推荐，性能最优）：
```bash
ss -tlnp | grep -E ':(8000|5555|5432|6379)\>'
```

**方法 2：格式化分组输出**（结果更清晰）：
```bash
ports=(8000 5555 5432 6379)
ss -tlnp > /tmp/ports_snapshot.txt
for port in "${ports[@]}"; do
  echo "--- Port :$port ---"
  grep ":$port\>" /tmp/ports_snapshot.txt || echo "  ✓ 空闲"
done
rm /tmp/ports_snapshot.txt
```

**方法 3：并行查询**（大量端口时使用）：
```bash
ports=(8000 5555 5432 6379 3000 5173 8080)
printf '%s\n' "${ports[@]}" | xargs -P 4 -I {} sh -c 'echo "Port {}: $(ss -tlnp | grep -q ":{}\>" && echo "占用" || echo "空闲")"'
```

> **性能对比**：检查 10 个端口，方法 1 耗时 ~10ms，原 for 循环耗时 ~100ms  
> **注意**：`\>` 用于精确匹配端口边界，避免 8000 匹配到 18000

## 查看进程与服务状态

查看所有 Docker 容器及端口映射：
```bash
docker ps --format "table {{.Names}}\t{{.Ports}}\t{{.Status}}"
```

查看 uvicorn（FastAPI）是否在运行：
```bash
ps aux | grep uvicorn | grep -v grep
```

查看 Celery Worker 是否在运行：
```bash
ps aux | grep celery | grep -v grep
```

查看服务器上所有已运行服务（systemd）：
```bash
systemctl list-units --type=service --state=running
```

## 释放端口 / 终止进程

获取占用端口的 PID（仅输出数字）：
```bash
sudo lsof -ti :8000
```

直接终止占用指定端口的进程：
```bash
sudo kill $(sudo lsof -ti :8000)
```

强制终止（进程不响应时使用）：
```bash
sudo kill -9 $(sudo lsof -ti :8000)
```

> `-9` 不给进程清理机会，可能导致数据丢失，谨慎使用
