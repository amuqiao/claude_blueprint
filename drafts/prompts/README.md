# Prompt 草稿说明

## 目录区别

- `wip/`：正在整理、还会继续修改的 prompt 草稿
- `next/`：当前主线候选 prompt，已可反复使用，但还没进入正式 `prompts/`
- `archived/`：不再继续维护、仅保留作历史参考的 prompt

`wip/` 下允许再按主题做一层轻量目录，例如：

- `architecture/`
- `backend/`
- `operations/`
- `writing/`

原则：

- 先做生命周期分层，再做少量主题分组
- 主题分组只解决“好找”和“好维护”，不要过度细拆
- `wip/` 根目录尽量只保留说明文件和临时整理笔记

## Prompt 文档格式

- 一个文件聚焦一个主题
- 同主题下可以有多个版本，也可以分多个阶段
- 真正给 AI 使用的内容，放在中间可直接复制的代码块里

## 同主题的常见组织方式

- 多版本：如 `简单版`、`详细版`、`严格审查版`
- 多阶段：如 `阶段 1`、`阶段 2`
- 也可以同时存在：同一主题下，先分版本，再在某些版本内继续分阶段
