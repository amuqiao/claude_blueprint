# Markdown 文档可视化增强 Prompt

> **文档职责**：沉淀“输出 Markdown 文档时增强可视化与阅读体验”的可复用 prompt。
> **适用场景**：需要把分析、方案、说明类内容整理成 Markdown 文档，希望避免概念堆砌、增强层次感和可读性时。
> **目标读者**：本仓库维护者。
> **维护规范**：如果这类 prompt 的输入边界和输出结构稳定，可迁入 `prompts/`；如果后续被更完整的 writing 能力吸收，再迁入 `drafts/prompts/archived/`。

## 简单版

```text
请把内容整理成更有阅读体验的 Markdown 文档。

要求：
1. 不要只是堆概念和长段落
2. 适当增加可视化表达，例如表格、列表、文本树、Mermaid 或结构化分段
3. 优先让读者快速看懂整体，再进入细节
4. 保持简洁，不要为了花哨而花哨
5. 遵循 writing 规范输出
```

## 详细版

```text
请把内容输出为更有阅读体验的 Markdown 文档，不要只是平铺直叙地堆砌概念。

要求：
1. 先让读者快速建立整体理解，再展开细节
2. 根据内容特点，主动选择更合适的表达形式，例如：
   - 表格：用于对比、分类、职责边界
   - 列表：用于步骤、清单、要点
   - 文本树：用于目录结构、层级关系
   - Mermaid：用于流程、关系、架构
   - 分段小结：用于每一节的结论收束
3. 避免大段抽象论述连续出现，必要时切成更清晰的小节
4. 可适当补“你应该先看什么 / 当前重点是什么 / 一句话结论”这类导读
5. 图示和结构化表达要服务理解，不要为了可视化而可视化
6. 输出仍然保持正式、克制、易维护
7. 遵循 writing 规范输出
```

## 强化版

```text
请把内容整理成“更适合阅读和理解”的 Markdown 文档。

不要只把信息写全，还要主动优化阅读体验。

请优先做这些事：
1. 先给结论或导读，降低进入门槛
2. 将连续大段文字拆成更有层次的结构
3. 遇到抽象概念时，优先考虑用表格、流程图、结构图或清单辅助理解
4. 对重点部分做视觉分层，让读者能快速扫描
5. 如果某部分天然适合可视化，请直接补合适的 Markdown 表达，不要只停留在文字说明
6. 保持整体风格专业、简洁，不要堆砌装饰性内容
7. 遵循 writing 规范输出
```

## 当前判断

- 这类 prompt 的核心不是“做漂亮排版”，而是“让 Markdown 更易读、更易扫、更易理解”
- 当前更适合作为 prompt 资产，不急着直接升 skill
- 后续如果你频繁需要提升文档表现力，可以继续提炼为正式 prompt，或并入 writing 相关能力
