# 批量 API 调用脚本 Prompt

> 状态：派生自真源 skill 的日常使用 prompt，不再作为唯一真源维护。

适用场景：
- 需要写一个“输入是一批任务，输出是每个任务文件产物”的 Python 脚本
- 核心难点在并发、重试、文件输出和报告整理

```text
请帮我编写一个“批量 API 调用脚本”。

这个脚本的特点是：
- 输入是一批任务
- 核心操作是对外部 API 发请求
- 每个任务最终要产出文件
- 需要考虑并发、失败重试、日志和报告

要求：
1. 先按清晰结构组织脚本，不要上来把所有逻辑混在 main() 里。
2. 至少明确这些层次：
   - Config：配置
   - Task / Job / Result：数据结构
   - call_one：单次 API 调用
   - retry_call：重试包装
   - build_job_list：任务展开
   - write_report：报告写出
   - main：初始化与编排
3. 函数职责单一：
   - call_one 只负责单次调用
   - retry_call 只负责重试
   - run_jobs / main 负责编排
   - write_report 只负责写报告
4. 输出目录、日志、报告和 dry-run 行为要明确。
5. 如果需要并发，优先先给出清晰的并发策略和错误处理方式，再写实现。
6. 代码里不要混入和任务无关的工程方法论；只围绕这次脚本本身展开。

我的任务背景如下：
【把输入、输出、API、并发需求、文件产物需求贴在这里】
```

## 自检清单

- [ ] 是否明确区分了 Config、Task、Job、Result
- [ ] `call_one` 是否只做单次 API 调用
- [ ] 是否把重试逻辑独立出来，而不是散落在主流程里
- [ ] 是否明确了 dry-run 行为
- [ ] 是否明确了输出目录、日志和报告的组织方式
- [ ] 是否说明了并发策略和失败后的处理方式
- [ ] 是否避免把业务参数散装传来传去
- [ ] `main()` 是否只负责初始化、校验和编排
