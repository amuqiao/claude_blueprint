# 方法论文档可视化增强 Prompt

> **文档职责**：沉淀“方法论文档如何增强可读性、结构感和可视化表达”的可复用 prompt。
> **适用场景**：需要输出方法论、范式、分层模型、能力边界、认知框架等文档，希望减少概念堆砌、增强整体理解时。
> **目标读者**：本仓库维护者。
> **维护规范**：如果 writing 的 rule / skill 边界调整，或这类增强 prompt 的输入输出结构稳定变化，需要同步更新本文档。

## 定位

这类 prompt 的作用是“增强表达效果”，不是替代基础写作规范。

边界：

- 文档头部模板、表达底线、图示使用边界，遵循 `rules/writing.md`
- 通用写作结构、重构逻辑和基础自检，优先参考同目录的 `新建正式文档 Prompt.md`
- 本 prompt 只负责把“方法论文档”的结构化表达和可视化表达再拉高一档

## 标准版

```text
请把内容整理成更适合阅读的方法论文档。

要求：
1. 先帮助读者建立整体理解，不要一上来堆概念
2. 优先讲清楚主干结构、层级关系、边界区别
3. 对高抽象内容，主动改写成更容易扫描的结构，不要连续保留大段抽象论述
4. 如适合，补充对比表、结构化清单、文本树或 Mermaid 图
5. 图示必须服务理解，不要为了“看起来丰富”而增加装饰
6. 保持专业、克制、美观
7. 输出为正式 Markdown 文档，遵循 writing 规范
```

## 强化版

```text
请把内容输出为“更适合阅读和理解”的方法论文档。

要求：
1. 开头先给一句核心结论，或先给一个整体理解框架
2. 优先按这个顺序组织内容：
   - 整体理解
   - 主干结构
   - 层级关系
   - 边界区别
   - 常见误区 / 例外
   - 快速参考
3. 当内容涉及分层、边界、对比、优先级、依赖关系时，优先考虑：
   - 对比表
   - 结构化清单
   - 文本树
   - Mermaid 图
4. 如果某段话抽象度太高，请主动重写结构，不要只做措辞润色
5. 图和表必须优先表达主干结构、层级关系、边界区别
6. 保持专业、克制，不做花哨装饰
7. 输出为正式 Markdown 文档，遵循 writing 规范
```

## 适合使用的场景

- 方法论整理
- 心智模型说明
- 分层模型讲解
- 文档体系与职责边界说明
- 能力地图、能力边界、能力关系说明

## 不适合使用的场景

- 只是补一小段说明
- 纯命令参考或纯接口参数表
- 已经非常具体、几乎不含抽象结构的操作文档

## 当前判断

- 这类 prompt 更适合作为 writing 能力的“增强开关”，而不是底层规范
- 它适合在高抽象、高结构密度的文档场景中使用
- 后续如果这类增强场景继续稳定，可以再考虑迁入正式 `prompts/`
