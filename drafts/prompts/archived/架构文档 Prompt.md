# 架构文档 Prompt

> 状态：派生自真源 skill 的日常使用 prompt，不再作为唯一真源维护。

适用场景：
- 为整个项目写一份架构文档
- 目标是明确技术骨架、分层方式、目录结构和核心业务流

```text
请为这个项目整理一份“架构文档”。

要求：
1. 这是一份项目级架构文档，只写骨架，不写模块实现细节。
2. 至少覆盖：
   - 技术选型：选了什么，为什么选
   - 目录结构：目录层级 + 每个目录所属层
   - 核心业务流：哪个层做了什么、层与层之间如何流转
3. 不要写：
   - 具体文件名清单
   - 某个模块的内部算法
   - 具体 API 签名
4. 如果存在前后端分层，请分别说明：
   - 每层职责
   - 每层禁止做的事
5. 如果合适，用文本树或 Mermaid 表达目录结构和核心业务流。

项目背景如下：
【把项目背景、技术栈、目录、已有思路贴在这里】
```

## 自检清单

- [ ] 是否只写骨架，不写模块实现细节
- [ ] 技术选型是否只写结论和理由
- [ ] 目录结构是否映射到分层
- [ ] 是否说明了各层职责与禁止事项
- [ ] 核心业务流是否讲清层间流转
