# 架构文档可视化增强 Prompt

> **文档职责**：沉淀“架构设计文档如何增强可视化与阅读体验”的可复用 prompt。
> **适用场景**：需要输出或补强架构设计文档，希望减少纯文字堆砌、增强图示和结构表达时。
> **目标读者**：本仓库维护者。
> **维护规范**：如果这类 prompt 的输入边界和输出结构稳定，可迁入 `prompts/`；如果后续被更完整的架构文档能力吸收，再迁入 `drafts/prompts/archived/`。

## 标准版

```text
请把内容整理成更适合阅读的架构文档。

要求：
1. 先让读者快速看懂系统整体结构，再进入细节
2. 适当增加可视化表达，例如整体结构图、分层图、核心链路图、组件关系图、职责对比表
3. 不要只是列目录和模块名，要让读者知道它们之间如何协作
4. 避免长段概念堆砌，优先突出系统结构、关键依赖和核心调用链
5. 图示要服务理解，优先画最有价值的图，不要为了完整而堆图
6. 遵循 writing 规范输出
```

## 强化版

```text
请把内容输出为更适合阅读和维护的架构文档。

要求：
1. 先给整体结构导读，再展开具体模块和技术细节
2. 根据内容特点，优先考虑补这些表达形式：
   - 整体结构图
   - 分层能力结构图
   - 核心组件关系图
   - 核心链路图
   - 技术栈表或部署说明表
3. 不要只写“有哪些模块”，要讲清：
   - 模块职责
   - 调用关系
   - 依赖方向
   - 哪些是当前真实实现，哪些是后续扩展
4. 如果某部分适合可视化，请直接补 Mermaid 初稿或结构化表格
5. 保持正式、克制、可维护，不做装饰性排版
6. 遵循 writing 规范输出
```

## 当前判断

- 这类 prompt 更适合整体架构、分层结构、组件关系、核心链路等架构类内容
- 当前更适合作为 prompt 资产，不急着直接升 skill
