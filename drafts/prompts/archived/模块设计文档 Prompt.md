# 模块设计文档 Prompt

> 状态：派生自真源 skill 的日常使用 prompt，不再作为唯一真源维护。

适用场景：
- 为某个具体功能模块写设计文档
- 目标是形成该模块的唯一设计基线

```text
请为下面这个功能模块整理一份“模块设计文档”。

要求：
1. 这是模块设计文档，不是架构总览文档。
2. 文档至少按下面结构组织：
   - 问题定义
   - 目标与非目标
   - 方案设计
   - 数据模型（有 DB 改动时必须）
   - 后端接口规范（有接口时必须）
   - 前端集成与交互协议（有复杂交互时必须）
   - 备选方案
   - 待定事项（有则必须）
3. 方案设计里要讲清：
   - 核心流程
   - 与上下游模块的边界
   - 关键设计决策
4. 数据模型需要完整字段定义；枚举值要列全。
5. 接口规范至少包含：
   - 基本信息
   - 请求体
   - 响应体
   - 错误码
6. 如果有前端页面，不要把 UI 布局细节混进正文；正文只讲交互协议和状态流转。

模块背景如下：
【把模块背景、需求、约束、已有方案贴在这里】
```

## 自检清单

- [ ] 目标是否可验收，非目标是否明确
- [ ] 核心流程是否清楚，边界是否明确
- [ ] 数据模型是否完整，枚举值是否列全
- [ ] 接口规范是否包含基本信息、请求体、响应体、错误码
- [ ] 有复杂交互时，是否补了前端集成与交互协议
- [ ] 是否给出至少 2 个备选方案
- [ ] 待定事项是否是真实未决问题
- [ ] 有前端页面时，是否同步考虑 `mock.html`
