# 运维诊断 CLI Prompt

> 状态：派生自真源 skill 的日常使用 prompt，不再作为唯一真源维护。

适用场景：
- 需要写一个面向终端使用的 Python 运维诊断工具
- 需要以子命令组织只读诊断和可选写入操作

```text
请帮我编写一个“运维诊断 CLI 工具”。

这个工具的目标是：
- 通过子命令组织不同诊断动作
- 支持只读诊断
- 如有写入动作，默认 dry-run，必须显式 --apply
- 可能涉及 DB、Redis、进程状态、日志等多数据源

要求：
1. 结构上采用“一个主脚本 + 多个子命令”的方式。
2. 子命令之间不要互相调用，每个子命令自己完成自己的诊断或修复动作。
3. 只读诊断和写入动作要严格区分。
4. 写入动作必须默认安全：
   - 不加 --apply 只打印将执行什么
   - 加 --apply 才真正写入
5. 数据访问要分层：
   - DB 查询
   - 外部服务访问
   - 进程状态读取
   - 富化整合
6. 输出直接面向终端阅读，不要默认生成报告文件。
7. 命令帮助、使用方式和输出样式要清楚，适合一线排障时直接执行。

我的诊断目标如下：
【把诊断目标、涉及的数据源、预期子命令贴在这里】
```

## 自检清单

- [ ] 是否采用子命令结构，而不是一个巨型入口函数
- [ ] 只读诊断和写入动作是否明确区分
- [ ] 写入动作是否默认 dry-run，且需要 `--apply`
- [ ] DB / 外部服务 / 进程信息 / 富化逻辑是否分层
- [ ] 输出是否直接适合在终端阅读
- [ ] 是否提供了清楚的使用方式和 help
- [ ] 子命令之间是否避免互相调用
- [ ] 是否避免把项目特有常量硬编码成通用规则
