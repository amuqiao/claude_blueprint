# WIP Prompt 草稿说明

本目录存放还在打磨、但已经值得继续沉淀的 prompt 草稿。

当前按“主题”做一层轻量分组，不再把所有 prompt 平铺在根目录：

- `architecture/`：需求分析、架构设计、从方案到实现、实现后回写文档
- `backend/`：后端接口、测试、验证这类实现侧 prompt
- `operations/`：部署、服务管理、安全、配置治理、开发环境检查
- `writing/`：文档输出、讲解、可视化增强、接口说明

根目录只保留：

- 目录说明文件
- 临时整理笔记，如 `note.md`

维护规则：

- 优先先按主题归类，不要继续在根目录平铺新增 prompt
- 主题数量控制在少量稳定入口，够用即可，不追求细分类
- 只有同一主题下文件继续明显增多时，才考虑在该主题目录下再细分
