# architecture

放“需求分析、架构设计、实现前后衔接”这类 prompt。

典型内容：

- 通用版：
  - `架构文档 Prompt.md`
  - `模块设计文档 Prompt.md`
- 需求分析与技术栈选型
- 架构方案质检
- 从粗架构推进到实现
- 基于已实现代码补强架构文档

组织约定：

- `通用版`：直接面向“写架构文档 / 写模块设计文档 / 做文档自检”这类高频基础任务。
- `专项版`：面向特定阶段或特定输入，例如“从粗架构推进到实现”“从已实现代码反推架构文档”“对架构方案做质检”。
- 如果两个 prompt 看起来相似，优先保留一份 `通用版`，再让 `专项版` 负责额外阶段信息，而不是重复写同一套基础结构。

这里不放：

- 纯后端实现细节
- 部署、安全、配置治理
- 单纯的文档表达与可视化增强
