# backend

放“后端实现与接口验证”这类 prompt。

典型内容：

- 通用版：
  - `FastAPI 后端 Prompt.md`
  - `批量 API 调用脚本 Prompt.md`
- Request Body 示例补齐
- FastAPI 单元测试补齐
- 接口改动后的验证与报告

组织约定：

- `通用版`：解决“FastAPI 后端怎么起盘 / 怎么评审 / 批处理脚本怎么写”这类高频基础任务。
- `专项版`：解决更窄的实现问题，例如 request body 示例、接口验证、单测补齐。
- 如果一个 prompt 只是某个专项场景的变体，不要重复维护一整套通用规则，优先复用 `通用版` 的约束。

这里不放：

- 项目级架构设计
- 部署与服务管理
- 面向读者的正式文档输出
