# operations

放“部署、服务运行、环境检查、安全、配置治理”这类 prompt。

典型内容：

- 通用版：
  - `运维诊断 CLI Prompt.md`
  - `服务生命周期管理脚本 Prompt.md`
- 部署与服务管理
- Docker 开发环境检查
- dev.sh 总控脚本
- 服务安全检查
- 项目配置治理检查

组织约定：

- `通用版`：解决“如何写诊断 CLI / 如何写服务管理脚本 / 怎么做基础自检”。
- `专项版`：解决某个具体运维场景，例如端口检查、配置治理、安全检查。
- 如果一个 prompt 更像“这类问题该放哪里、怎么归档”，优先考虑放到 `drafts/docs/wip/`，而不是继续堆进运行层。

这里不放：

- 需求分析与架构主线设计
- 纯后端接口实现
- 文档写作与表达增强
