# 服务生命周期管理脚本 Prompt

适用场景：
- 需要为项目编写统一的 `dev.sh` / `service.sh` 类服务管理脚本
- 目标是统一管理多个进程的 start / stop / restart / status / logs

```text
请帮我编写一个“服务生命周期管理脚本”。

目标：
- 用一个脚本统一管理多个服务
- 提供稳定的 start / stop / restart / status / logs 命令
- 清楚管理 PID、日志、端口冲突和输出格式

要求：
1. 一个项目只有一个总控脚本，不要为每个服务单独造命令入口。
2. 配置与逻辑分离：
   - 服务名列表
   - 日志路径
   - PID 文件路径
   - 端口
   - URL
   这些都应放在配置函数里。
3. 标准命令契约固定：
   - start
   - stop
   - restart
   - status
   - logs
4. 进程控制要明确：
   - 启动后做存活检测
   - 停止时先 SIGTERM，再按需 SIGKILL
   - stale PID 自动清理
5. 端口冲突要明确报错，不允许静默失败。
6. 日志和终端输出要可读、可扫。
7. 如果项目有多个服务，输出时要保证服务名列对齐。

我的服务列表和运行需求如下：
【把服务名、启动命令、端口、日志目录、URL 贴在这里】
```

## 自检清单

- [ ] 是否只有一个总控脚本统一管理所有服务
- [ ] 服务配置和核心逻辑是否分离
- [ ] 是否实现了 `start / stop / restart / status / logs`
- [ ] `logs` 是否要求明确的服务名
- [ ] 是否有 PID 文件管理和 stale PID 清理
- [ ] 启动后是否做存活检测
- [ ] 停止时是否先 SIGTERM，再按需 SIGKILL
- [ ] 端口冲突时是否明确报错且不中断其他服务的处理
- [ ] 日志路径和终端输出是否清楚
- [ ] 多服务输出时是否做了列对齐
