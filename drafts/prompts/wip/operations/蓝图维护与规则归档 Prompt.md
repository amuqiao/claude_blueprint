# 蓝图维护与规则归档 Prompt

适用场景：
- 需要修改这套 blueprint 本身
- 需要判断一条新约束该放到哪里
- 需要决定某个规范应该进运行层、项目层，还是文档层

```text
我现在要维护这套 blueprint，请不要直接修改文件，先帮我判断“这条新约束 / 新规范 / 新经验”应该放在哪一层。

请按下面顺序判断：
1. 这件事是在改变 AI 的行为方式，还是在记录项目/仓库决策？
2. 如果是在改变 AI 行为：
   - 它是跨项目长期成立的吗？
   - 还是只在当前项目或当前仓库成立？
3. 如果是跨项目长期成立：
   - 它是 1 到 3 行的硬约束，还是已经需要一整套模板/说明？
4. 如果它不是运行层约束，而是方法、经验、判断标准：
   - 更适合放 docs
   - 还是更适合放 drafts/prompts

输出要求：
1. 先给归属结论。
2. 再说明为什么应放在这一层。
3. 再说明不应该放到哪些层，以及原因。
4. 如适合，最后给出建议文件路径。

待判断内容如下：
【把新约束、新经验或想新增的规则贴在这里】
```
