# 蓝图维护与规则归档 Prompt

> 状态：日常使用 prompt，服务于 `drafts/` 层的收口与规则归档，不属于 `personal-os` 真源本体。

> **文档职责**：提供一个日常可直接使用的快捷 prompt，用于维护 blueprint、判断规则归属、收口旧规范和做月度回顾。
> **适用场景**：需要快速发起“真源放哪层 / 这个东西要不要进 skill / 这些 prompt 该不该归档”这类任务时。
> **目标读者**：本仓库维护者。
> **维护规范**：这份 prompt 只服务日常判断和草稿收口；如果某条方法被长期验证稳定，再决定是否补进 `skills/personal-os/`。

## 标准版

```text
请帮我判断这条新约束 / 新规范 / 新经验应该放在哪一层。

请按下面顺序处理：
1. 先判断这是在改变 AI 行为，还是在记录项目事实 / 决策
2. 再判断它属于真源层、使用层，还是事实层
3. 如果它需要长期维护，再判断它更适合进：
   - skills/
   - drafts/prompts/
   - docs/
4. 最后判断它是否真的值得升成 skill，还是只应保留为日常 prompt

输出要求：
1. 先给归属结论
2. 再说明判断依据
3. 再说明不该放到哪些层，以及原因
4. 如适合，给出建议文件路径

待判断内容如下：
【把新约束、新经验或想新增的规则贴在这里】
```

## 严格审查版

```text
请严格审查这条规则 / 经验是否值得长期沉淀。

请重点检查：
1. 它是不是跨项目稳定成立
2. 它到底是在改变 AI 行为，还是只是在记录当前项目事实
3. 它更适合进 skill、prompt、docs，还是根本不该保留
4. 它是否已经和现有真源或现有 prompt 重复
5. 如果保留，唯一真源应该放在哪里

输出格式：
1. 总体结论
2. 推荐归属位置
3. 不推荐的位置与原因
4. 是否值得长期保留
5. 如果保留，建议文件路径

待审查内容如下：
【把规则、经验、已有文档或草稿贴在这里】
```

## 月度回顾版

```text
请把这次任务当成一次 blueprint 月度回顾。

请按下面顺序检查：
1. 哪些规则现在还在多处维护
2. 哪些 prompt 已经不该继续当主维护位
3. 哪些内容应该回收进 skills/
4. 哪些内容仍然适合保留在 drafts/prompts/
5. 当前最值得做的 1 到 3 个收口动作是什么

输出要求：
1. 先给总体判断
2. 再列问题清单
3. 再列优先级最高的维护动作
4. 不要为了显得全面而硬提无关动作
```
