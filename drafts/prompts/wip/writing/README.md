# writing

放“文档输出与表达类” prompt，例如项目讲解、接口文档、请求示例、可视化增强。

推荐分成两类理解：

- 通用版：
  - `新建正式文档 Prompt.md`
  - `代码讲解文档 Prompt.md`
- 专项版：
  - 接口文档
  - 请求示例
  - 可视化增强
  - 项目讲解

组织约定：

- `通用版` 解决“怎么写一篇更好的文档”。
- `专项版` 解决“某一类特定文档怎么写得更合适”。
- 如果后续再迁 prompt，优先先判断它是通用写作能力，还是某个专项场景，不要两边重复维护。
- 明显离题、重复副本或已被通用版覆盖的文件，不继续留在 `wip/`。

当前已知噪音治理原则：

- `copy` 文件不留在 `wip/`
- 与当前 blueprint 主线无关的提示词不留在 `wip/`
- 已被通用版覆盖的历史专项 prompt 迁入 `drafts/prompts/archived/`

这里不放部署、安全、测试、配置治理这类非写作主题 prompt。
