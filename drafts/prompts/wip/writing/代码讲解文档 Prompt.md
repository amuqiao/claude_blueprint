# 代码讲解文档 Prompt

适用场景：
- 需要为某段代码、某个函数、某个模块写讲解文档
- 目标读者需要理解“为什么这样写”和“关键逻辑怎么工作”

```text
请为下面这段代码整理一篇“代码讲解文档”。

要求：
1. 只讲难点、混淆点、关键设计；命名已经自解释的简单逻辑不要硬讲。
2. 第一节必须是“心智模型”，先解释读者理解后续内容所需的认知前提。
3. 后续每个讲解单元按下面顺序判断并输出：
   - 问题段（可选）：为什么需要这段代码
   - 原理段（必须）：必须至少给出一种可视化表达，优先公式 / 示意图思路 / 伪代码
   - 代码段（必须）：只贴关键 3 到 10 行，不要粘贴整个函数
   - 参数段（有参数时必须）：说明参数含义，必要时补调参建议
4. 如果多个单元之间存在固定顺序依赖，最后补“执行顺序”一节。
5. 讲解重点放在 WHY，不要只是把代码翻译成自然语言。

输出结构建议：
- 心智模型
- 关键逻辑单元 1
- 关键逻辑单元 2
- ...
- 执行顺序（按需）

代码如下：
【把代码和必要上下文贴在这里】
```

## 自检清单

- [ ] 是否只讲难点、关键设计和混淆点
- [ ] 第一节是否为“心智模型”
- [ ] 每个讲解单元是否优先解释 WHY
- [ ] 原理段是否包含至少一种可视化表达思路
- [ ] 代码段是否只保留关键 3 到 10 行
- [ ] 参数存在时，是否说明参数含义和调参建议
- [ ] 多个单元存在顺序依赖时，是否补了“执行顺序”
