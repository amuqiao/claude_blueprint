# 文档可视化增强 Prompt

> **文档职责**：沉淀“在正式文档输出中增强可视化表达与阅读体验”的可复用 prompt。
> **适用场景**：需要输出 Markdown 文档、方法论文档或架构文档，希望减少概念堆砌、增强结构感与可读性时。
> **目标读者**：本仓库维护者。
> **维护规范**：本 prompt 是日常使用层入口；文档表达真源以 [skills/personal-os/references/document-writing.md](/Users/admin/Downloads/Code/claude_blueprint/skills/personal-os/references/document-writing.md:1) 为准，不在这里反向维护写作规则本体。

## 定位

这份 prompt 处理的是“增强表达效果”，不是替代基础写作规范。

边界：

- 文档表达真源：`skills/personal-os/references/document-writing.md`
- 本 prompt 只负责在日常任务中快速打开“更易读、更易扫、更有结构”的写法
- 如果写作真源和这里冲突，以真源为准

## 通用版

```text
请把内容整理成更有阅读体验的 Markdown 文档。

要求：
1. 不要只是堆概念和长段落
2. 适当增加可视化表达，例如表格、列表、文本树、Mermaid 或结构化分段
3. 优先让读者快速看懂整体，再进入细节
4. 保持简洁，不要为了花哨而花哨
5. 遵循 personal-os 下文档表达真源的规则输出
```

## 方法论文档增强版

```text
请把内容整理成更适合阅读的方法论文档。

要求：
1. 先帮助读者建立整体理解，不要一上来堆概念
2. 优先讲清楚主干结构、层级关系、边界区别
3. 对高抽象内容，主动改写成更容易扫描的结构，不要连续保留大段抽象论述
4. 如适合，补充对比表、结构化清单、文本树或 Mermaid 图
5. 图示必须服务理解，不要为了“看起来丰富”而增加装饰
6. 保持专业、克制、美观
7. 遵循 personal-os 下文档表达真源输出
```

适用场景：

- 方法论整理
- 心智模型说明
- 分层模型讲解
- 文档体系与职责边界说明
- 能力地图、能力边界、能力关系说明

## 架构文档增强版

```text
请把内容整理成更适合阅读的架构文档。

要求：
1. 先让读者快速看懂系统整体结构，再进入细节
2. 适当增加可视化表达，例如整体结构图、分层图、核心链路图、组件关系图、职责对比表
3. 不要只是列目录和模块名，要让读者知道它们之间如何协作
4. 避免长段概念堆砌，优先突出系统结构、关键依赖和核心调用链
5. 图示要服务理解，优先画最有价值的图，不要为了完整而堆图
6. 遵循 personal-os 下文档表达真源输出
```

## 不适合使用的场景

- 只是补一小段说明
- 纯命令参考或纯接口参数表
- 已经非常具体、几乎不含抽象结构的操作文档
- 只做局部措辞润色，不涉及结构与表达优化
