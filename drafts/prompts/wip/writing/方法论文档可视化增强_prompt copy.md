# 方法论文档可视化增强 Prompt

> **文档职责**：沉淀“方法论文档如何增强可视化与阅读体验”的可复用 prompt。
> **适用场景**：需要输出方法论、范式、分层模型、能力边界等文档，希望减少抽象堆砌、增强结构感时。
> **目标读者**：本仓库维护者。
> **维护规范**：如果这类 prompt 的输入边界和输出结构稳定，可迁入 `prompts/`；如果后续被更完整的 writing 能力吸收，再迁入 `drafts/prompts/archived/`。

## 标准版

```text
请把内容整理成更适合阅读的方法论文档。

要求：
1. 不要只是堆概念，要先帮助读者建立整体理解
2. 适当增加可视化表达，例如分层图、关系图、对比表、结构化清单
3. 优先把“主干结构、层级关系、边界区别”讲清楚
4. 对抽象概念，尽量用更容易扫描的方式表达
5. 保持专业、克制，不做花哨装饰
6. 遵循 writing 规范输出
```

## 强化版

```text
请把内容输出为“更适合阅读和理解”的方法论文档。

要求：
1. 先给一句核心结论或主干模型，降低理解门槛
2. 对方法、范式、分层、能力边界等内容，优先用：
   - 分层图
   - 对比表
   - 主干流程图
   - 分类清单
3. 不要连续输出大段抽象论述；遇到概念密度高的部分，要主动切成更易读的结构
4. 如适合，可补一张 Mermaid 图或文本树，帮助读者快速建立脑图
5. 图示和结构化表达必须服务理解，不要为了视觉效果而增加复杂度
6. 遵循 writing 规范输出
```

## 当前判断

- 这类 prompt 更适合方法论、认知模型、文档体系、能力边界等高抽象内容
- 当前更适合作为 prompt 资产，不急着直接升 skill
