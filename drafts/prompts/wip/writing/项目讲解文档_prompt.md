# 项目讲解文档 Prompt

> **文档职责**：沉淀“项目讲解文档”类 prompt，供后续打磨和提炼为正式 prompt 或 skill。
> **适用场景**：第一次接触一个项目，需要快速产出面向开发者的项目导览文档。
> **目标读者**：本仓库维护者。
> **维护规范**：如果这个 prompt 的输入边界、输出结构或适用场景稳定下来，迁入 `prompts/`；如果被 skill 吸收，迁入 `drafts/prompts/archived/`。

## 项目总览版

```text
请帮我写一份“项目讲解文档”，面向第一次接触该项目的开发者。

要求：
1. 先给项目整体定位，再讲目录结构和模块分工
2. 说明核心运行链路，而不是只列目录
3. 讲清技术栈、分层结构、关键模块职责
4. 对重要代码路径给出“从入口到落地”的说明
5. 输出为正式 Markdown 文档，遵循 writing 规范
6. 如果项目过大，先给总览版，再标出后续可以继续展开的部分
```

## 模块讲解版

```text
请帮我写一份“模块讲解文档”，面向第一次维护该模块的开发者。

要求：
1. 先讲模块职责和边界
2. 再讲目录结构与关键文件
3. 说明核心调用链和数据流
4. 标出最容易误解的设计点
5. 输出为正式 Markdown 文档，遵循 writing 规范
```

## 当前判断

- 这类 prompt 已经具备复用价值
