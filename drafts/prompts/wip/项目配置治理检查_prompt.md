# 项目配置治理检查 Prompt

> **文档职责**：沉淀“检查项目配置治理、消除硬编码并给出统一配置方案”的可复用 prompt。
> **适用场景**：需要检查项目中的硬编码、环境变量、集中配置、`.env` / `.env.example`、Docker 配置和加载优先级时。
> **目标读者**：本仓库维护者。
> **维护规范**：如果这类 prompt 的输入边界和输出结构稳定，可迁入 `prompts/`；如果后续被更完整的配置治理能力吸收，再迁入 `drafts/prompts/archived/`。

## 简单版

```text
请帮我检查这个项目的配置治理是否合理。

要求：
1. 先检查是否存在硬编码配置
2. 判断配置是否应该集中管理
3. 检查是否需要 `.env`、`.env.example`、统一配置模块
4. 检查 `.env`、Docker、启动脚本之间是否存在覆盖冲突
5. 最后给出整改建议和推荐的配置加载优先级
5. 不要直接修改代码，先分析再给方案
```

## 详细版

```text
请对这个项目做一次“配置治理检查”，重点关注硬编码、环境变量、集中配置、配置来源一致性和加载优先级。

请按下面顺序分析：

1. 硬编码检查
- 检查代码中是否存在写死的配置项
- 例如：数据库地址、Redis 地址、API Key、端口、第三方服务 URL、调试开关、路径配置等
- 指出这些硬编码分别在哪些文件、哪些位置

2. 配置集中化检查
- 判断当前项目是否已经有统一配置入口
- 如果没有，说明哪些配置应该收口到统一 settings/config 模块
- 如果有，检查是否仍存在绕过统一配置入口的情况

3. 环境变量与文件治理
- 判断是否应该使用 `.env`
- 判断是否应该提供 `.env.example`
- 检查 `.env.example` 应包含哪些字段
- 检查哪些配置不应提交真实值

4. 配置来源枚举
- 枚举当前项目有哪些配置来源
- 至少检查：
  - 代码默认值
  - `.env`
  - `.env.example`
  - 系统环境变量
  - Dockerfile
  - docker-compose.yml / compose.yaml
  - 启动脚本
  - CI/CD secrets 或部署平台环境变量
- 判断这些配置来源是否职责清楚，还是已经相互混杂

5. Docker / 部署配置一致性
- 检查 Dockerfile、docker-compose、启动脚本、CI 配置是否与环境变量方案一致
- 判断是否存在一部分从 `.env` 读取、一部分写死在 Docker 配置中的问题
- 判断是否存在 `.env` 已配置，但运行时实际被 Docker / compose / 启动脚本覆盖的情况

6. 配置加载优先级
- 请给出推荐的优先级方案
- 明确：
  - 哪些配置源只是“默认值”
  - 哪些配置源可以覆盖 `.env`
  - 哪些配置源优先级最高
- 例如：代码默认值 / `.env` / docker-compose env / 系统环境变量 / CI secrets
- 说明为什么这样设计
- 特别指出：如何避免“用户在 `.env` 里配置了，但实际运行时被 Docker 配置覆盖却不自知”

输出要求：
1. 先给总体判断
2. 再列主要问题
3. 每个问题按“位置 / 风险 / 建议”展开
4. 最后给一份“推荐配置治理方案”
5. 如果适合，请补一份推荐的配置优先级说明
6. 如果发现存在覆盖链混乱，请明确画出当前覆盖关系和推荐覆盖关系

补充要求：
- 不要只指出“应该用环境变量”，要结合当前项目结构，明确哪些配置该收口、该怎么加载、谁优先级更高
```

## 强治理版

```text
请把自己当成项目配置治理审查人，对这个项目做一次系统检查。

目标：
- 消除硬编码
- 统一配置入口
- 建立 `.env` / `.env.example` 规范
- 让本地开发、Docker、生产部署的配置加载方式一致
- 明确配置优先级与覆盖关系

请重点检查：
1. 是否存在硬编码配置
2. 是否缺少统一配置模块
3. `.env` / `.env.example` 是否缺失或不完整
4. Dockerfile / docker-compose / 启动脚本 / CI 是否与配置方案割裂
5. 是否存在 `.env` 与 Docker / 启动脚本互相覆盖但边界不清的问题
6. 是否缺少一份明确的配置加载优先级规则

输出格式：
1. 总体判断
2. 高优先级问题
3. 中优先级问题
4. 推荐配置治理方案
5. 推荐的配置加载优先级
6. 当前覆盖链 vs 推荐覆盖链
7. 如需要，再补一个 `.env.example` 字段清单
```
