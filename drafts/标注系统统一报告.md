# 标注系统统一报告

**日期**: 2026-05-12  
**任务**: 统一目录结构参考文档的标注系统

---

## 一、问题描述

### 原标注系统（混乱）

在修正参考文档时，引入了**7 种不同的标注**：

1. 【官方确认】
2. 【官方文档确认】
3. 【官方支持，需手动创建】
4. 【官方支持，按需创建】
5. 【官方支持，使用时自动创建】
6. 【运行时生成】
7. 【安装插件后生成】

**问题**：
- ❌ 标注过多，用户难以区分
- ❌ 含义重叠，没有清晰边界
- ❌ 表述冗长，影响阅读体验
- ❌ 不一致，降低文档专业性

---

## 二、统一方案

### 新标注系统（清晰）

定义**三级标注系统**：

#### 【官方标准】
**含义**: 官方文档明确定义的核心目录/文件，是标准结构的一部分；不等于 blueprint 必须默认纳入

**适用于**:
- CLAUDE.md, settings.json
- commands/, skills/, agents/
- plugins/, projects/
- .mcp.json, .worktreeinclude

#### 【官方支持】
**含义**: 官方文档提及支持的功能，按需手动创建或使用时自动创建

**适用于**:
- keybindings.json
- rules/, output-styles/, agent-memory/

#### 【自动生成】
**含义**: Claude Code 运行时自动创建，无需用户手动管理

**适用于**:
- plugins/ 内的文件
- projects/ 内的文件
- 运行时目录（downloads/, sessions/ 等）

使用 `replace_all: true` 批量替换：
- 【官方确认】→ 【官方标准】

- 【官方支持，*】→ 【官方支持】
- 【运行时生成】→ 【自动生成】
- 【安装插件后生成】→ 【自动生成】
- 【不确定】→ 删除

## 八、后续维护

### 新增内容标注原则

判断流程：
1. **官方文档明确定义 + 属于核心结构位置** → 【官方标准】
2. **官方文档提及 + 需要创建或按需使用** → 【官方支持】
3. **运行时自动创建 + 不需要用户管理** → 【自动生成】

### 文档更新

当官方文档更新时：
1. 检查新增的目录/文件
2. 按判断流程确定标注
3. 更新两个参考文档
4. 同步更新 `标注系统说明.md`
5. 记录变更历史
