# 通用选型补盲与决策 Prompt

> **文档职责**：沉淀一套可复用的 prompt，用于在自己经验不足、概念模糊、看不见完整方案空间时，先补齐认知，再做选型判断。
> **适用场景**：遇到陌生主题、默认只想到一两种方案、还分不清概念边界、需要先扩展思路再做决策时。
> **目标读者**：本仓库维护者。
> **维护规范**：当前作为 `prompts/decision/` 下的母 prompt 维护；如果后续形成更稳定的方法体系，可继续拆分为子 prompt、模板和配套说明。

## 这类 Prompt 解决什么问题

这类 prompt 不是为了让模型替你直接拍板，而是先解决下面几个前置问题：

- 我脑子里的概念是否完整
- 我是不是只看到了 1 到 2 个熟悉方案
- 这个问题常见的方案空间到底有哪些
- 这些方案之间的差异、边界、适用场景是什么
- 在信息还不完整时，应该先问什么，再决定什么

它的价值不在于"直接给答案"，而在于：

- 帮你先看到常见方案
- 帮你补齐概念定义
- 帮你形成比较框架
- 帮你知道下一步该问什么

***

## 使用原则

- 先补盲，再决策
- 先枚举常见方案，再评价优劣
- 先定义概念，再进入推荐
- 先区分"适用场景"，再谈"哪个更好"
- 如果信息不足，先暴露不确定性，不强行下结论

***

## 简单版

适用场景：

- 我对一个主题还不熟
- 只想到少数几个方案
- 想先把常见思路补齐

```text
我正在思考一个选型/决策问题，但我怀疑自己的认知不完整，可能只看到了少数几个熟悉方案。

请不要直接替我拍板，而是先帮我做“认知补盲”。

请按下面顺序输出：
1. 这个问题通常在解决什么
2. 这个主题下常见的方案有哪些
3. 我目前可能遗漏了哪些常见方案或概念
4. 这些方案分别适合什么场景
5. 它们之间最关键的区别是什么
6. 如果现在就要做决策，最应该先确认哪几个问题

要求：
- 先补齐方案空间，再给建议
- 不要默认我已经知道行业常见概念
- 如果有术语，先用通俗语言解释
- 明确指出“新手最容易漏掉的方案”

我的问题是：
【把你的问题贴在这里】
```

## 标准版

适用场景：

- 我需要系统化理解一个选型问题
- 我不仅要答案，还要知道为什么
- 我想把结果整理成可复用的思路笔记

```text
请把这次对话当成一次“选型补盲 + 决策辅助”。

我当前的问题是：
【把你的问题贴在这里】

我的已知想法是：
【写下你目前想到的方案、概念或直觉。如果没有，就写“目前没有成型想法”】

请不要直接进入推荐方案，而是按下面顺序帮我展开：

1. 问题定义
   - 这个问题本质上在决定什么
   - 它通常受哪些约束影响
   - 新手最容易把什么混在一起

2. 概念澄清
   - 这个主题下有哪些核心概念
   - 这些概念之间的边界是什么
   - 哪些词容易被混用，但其实不是一回事

3. 方案空间展开
   - 常见方案至少列出 A / B / C
   - 如果还有常被忽略但有价值的方案，也请补充
   - 每个方案都说明：它是什么、适合什么、不适合什么

4. 方案比较
   - 从理解成本、实现成本、维护成本、扩展性、表达效果、适用场景等角度比较
   - 如果某些比较维度不适用，请明确说明

5. 对我当前问题的判断
   - 基于我提供的信息，当前更推荐哪种方向
   - 为什么
   - 这个推荐依赖哪些前提
   - 哪些信息一旦变化，推荐可能会变化

6. 待确认问题
   - 如果现在还不能稳妥决策，请列出最关键的待确认问题
   - 按优先级排序

输出要求：
- 先解释，再比较，再推荐
- 不要把“常见”直接等同于“最适合”
- 不要假设我已经知道隐含的标准做法
- 如果我提出的问题本身就太模糊，请先帮我重写问题
```

## 深度版

适用场景：

- 我想把一个陌生主题真正学明白
- 我希望模型不仅给方案，还帮我建立“判断框架”
- 我后面还想把结果整理进笔记或方法文档

```text
我希望你不要只回答“选哪个”，而是帮助我建立这个主题的决策框架。

主题是：
【把主题写在这里】

当前问题是：
【把具体问题写在这里】

我目前的理解是：
【写下你已有理解；如果没有，可以写“概念模糊，只想到少量方案”】

请按下面结构输出：

1. 这个主题的决策对象是什么
   - 到底在选什么
   - 常见误区是什么
   - 为什么新手容易看不全

2. 先补齐概念地图
   - 核心概念列表
   - 概念之间的关系
   - 容易混淆的概念对照

3. 展开完整方案空间
   - 主流方案
   - 过渡方案
   - 特定场景下才成立的方案
   - 每类方案的典型使用条件

4. 建立判断维度
   - 这个主题通常用哪些维度做判断
   - 哪些维度最重要
   - 哪些维度只在特定场景下重要

5. 输出比较表
   - 方案
   - 适用场景
   - 优点
   - 风险 / 代价
   - 新手容易踩的坑
   - 不适用场景

6. 对我的情况做决策建议
   - 当前推荐
   - 备选方案
   - 为什么不是其他方案
   - 如果我追求的是别的目标，推荐会如何变化

7. 反向补盲
   - 如果我是新手，在这个主题下还有哪些值得主动补的知识点
   - 下一步应该继续问什么，才能把认知补齐

要求：
- 目标不是给唯一正确答案，而是建立可迁移的判断能力
- 不要跳过“概念地图”和“方案空间”
- 如果某个方案不是最佳实践，但在某些现实约束下合理，也请说出来
- 如果我的问题其实混杂了两个不同层级的决策，请拆开说明
```

## 笔记整理版

适用场景：

- 我已经有一份零散草稿
- 草稿里混着问题、概念、候选方案和临时判断
- 我想把它整理成“思路决策文档”

```text
请把下面这份草稿整理成一份“思路决策文档”，而不是最终结论文档。

整理目标：
- 帮我补齐我还没意识到的常见方案
- 帮我澄清概念定义和边界
- 帮我把“问题、选项、比较维度、待确认项、阶段性倾向”分开
- 保留不确定性，不要强行写成最终拍板

请按下面结构重组：
1. 这个问题到底在决定什么
2. 相关核心概念
3. 常见方案空间
4. 比较维度
5. 当前草稿里已经提到的选项
6. 草稿里还缺失但应该补充的选项
7. 当前待确认问题
8. 阶段性倾向
9. 后续还应该继续追问什么

要求：
- 不要只润色文字，要重组信息结构
- 如果草稿里把不同层级的问题混在一起，请拆开
- 如果草稿默认跳过了一些行业常见选项，请明确补上
- 输出要适合长期维护

原始草稿如下：
【把草稿贴在这里】
```

## 高频提问模板

下面这些短句可以直接作为你每次补问时的追加指令：

- `我现在可能只看到了熟悉的 1 到 2 个方案，请帮我把常见方案空间补齐。`
- `不要急着推荐，请先告诉我这个主题下通常有哪些可选路径。`
- `请先解释这个概念在行业里通常怎么定义，再谈选型。`
- `请指出我当前理解里可能缺失的概念和默认选项。`
- `如果我是新手，最容易漏掉的 A/B/C 方案是什么？`
- `请把“概念澄清”和“方案比较”分开输出。`
- `请告诉我这个问题里，哪些是核心决策，哪些只是实现细节。`
- `如果现在还不适合拍板，请先列出最值得确认的问题。`

***

## 适合写成“思路决策文档”的输出结构

如果你希望模型输出结果直接能收进笔记，可以要求它按这个结构输出：

```text
# 主题

## 1. 这个问题在决定什么
## 2. 核心概念与边界
## 3. 常见方案空间
## 4. 比较维度
## 5. 当前场景下的推荐判断
## 6. 仍待确认的问题
## 7. 新手容易遗漏的点
## 8. 后续可继续追问的方向
```

## 自检清单

- [ ] 我是否先补齐了方案空间，而不是一上来就让模型拍板
- [ ] 我是否要求模型先解释概念，再比较方案
- [ ] 我是否明确让模型指出“我可能遗漏的常见方案”
- [ ] 我是否把“最终决策”和“阶段性倾向”区分开了
- [ ] 我是否要求模型列出待确认问题，而不是在信息不足时硬下结论
- [ ] 我是否在输出里保留了可复用的判断维度

***

## 当前判断

- 这不是单一领域 prompt，而是通用的“认知补盲 + 选型辅助” prompt
- 它适合作为很多专业 prompt 的前置步骤
- 当你对一个主题还没有形成完整概念地图时，应优先使用这一类 prompt，而不是直接问“哪个更好”
