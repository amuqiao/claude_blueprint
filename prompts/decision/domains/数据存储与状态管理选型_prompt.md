# 数据存储与状态管理选型 Prompt

> **文档职责**：沉淀一套更贴近“数据存储 / 状态划分 / 读写组织”场景的决策 prompt。
> **适用场景**：需要判断数据应该放哪里、状态如何分层、读写路径如何组织、缓存与持久化如何权衡时。
> **目标读者**：本仓库维护者。
> **维护规范**：作为 `decision` 目录下的领域子 prompt 维护；当数据库、缓存、前端状态、同步策略等子问题明显膨胀后，再拆出更细专题。

## 这个子 Prompt 解决什么问题

这类 prompt 主要解决下面几类常见困惑：

- 我知道需要存数据，但不知道该放数据库、文件、缓存，还是内存状态
- 我不清楚哪些是临时状态，哪些是业务状态，哪些必须持久化
- 我会直接问“用什么数据库”，但还没有先定义数据特征和访问模式
- 我不知道读写路径应该直接走主存储，还是加缓存、索引、派生层
- 我不清楚状态应该集中管理、局部管理，还是按边界拆开

---

## 常见概念

这类问题里，最容易混在一起的概念包括：

- **持久化数据**：需要跨进程、跨会话长期保留的数据
- **运行时状态**：只在当前运行过程中存在的状态
- **缓存**：为了加速读写而引入的临时副本
- **数据源**：系统里真正作为事实来源的存储位置
- **派生数据**：由主数据计算出来、可重建的数据
- **一致性要求**：数据是否必须实时一致、最终一致，还是允许短暂偏差
- **访问模式**：读多写少、写多读少、按键查询、全文搜索、聚合统计等

## 标准版

```text
请把这次对话当成一次“数据存储与状态管理选型”分析，不要急着直接推荐数据库或状态库。

我当前的问题是：
【把你的问题贴在这里】

我目前想到的方案是：
【写下你现在想到的存储方式、状态管理方式或数据库；如果没有，就写“目前只有模糊想法”】

请按下面顺序帮我展开：

1. 先定义这个问题到底在决定什么
   - 是在决定数据放在哪里
   - 是在决定状态怎么分层
   - 是在决定读写路径怎么组织
   - 还是在决定缓存与一致性策略
   - 如果我把几个层级混在一起了，请拆开说明

2. 先补齐常见方案空间
   - 这个场景下通常有哪些主流路径
   - 请至少列出 A / B / C
   - 如果存在新手容易漏掉但现实中很常见的过渡方案，也请补上

3. 澄清核心概念
   - 持久化数据、运行时状态、缓存、数据源、派生数据、一致性要求、访问模式分别是什么意思
   - 哪些概念经常被混用
   - 它们分别影响什么

4. 比较这些方案
   - 从复杂度、维护成本、一致性、扩展性、性能、开发心智负担等角度比较
   - 明确每种方案更适合什么，不适合什么

5. 结合我的场景给出建议
   - 当前更推荐什么方向
   - 为什么
   - 这个建议依赖哪些前提
   - 如果我的目标从“先跑起来”变成“长期稳定维护”，推荐会怎么变化

6. 如果现在还不适合拍板
   - 请列出最值得继续确认的 3 到 5 个问题

输出要求：
- 先补齐方案空间，再进入推荐
- 不要把“熟悉”直接当成理由
- 如果我跳过了数据特征、访问模式或一致性要求，请明确指出
- 如果我的问题其实跨了前端状态、后端存储、缓存策略多个层级，请拆开回答
```

## 状态分层版

```text
我正在判断一个系统里的状态应该如何分层和存放。

当前背景如下：
【把背景贴在这里】

请按下面结构输出：
1. 当前有哪些类型的状态
2. 哪些是临时状态，哪些是业务状态，哪些要持久化
3. 哪些状态应该局部管理，哪些应该集中管理
4. 是否需要缓存、同步或派生层
5. 当前场景更适合什么分层方式
6. 为什么
7. 后续规模变大时应该如何演进

要求：
- 把“状态位置”和“状态来源”分开
- 把“缓存”和“真实数据源”分开
- 如果更简单的状态模型已经足够，请明确指出
```

## 思路决策文档版

```text
请把下面这份关于“数据存储 / 状态管理 / 读写路径”的草稿，整理成一份“思路决策文档”。

不要把它写成最终定版方案，而是写成帮助我持续判断的文档。

请按下面结构输出：
1. 这个问题在决定什么
2. 核心概念与边界
3. 常见方案空间
4. 比较维度
5. 当前草稿已提到的方案
6. 当前草稿遗漏但值得补充的方案
7. 当前待确认问题
8. 阶段性倾向
9. 后续建议继续追问的方向

要求：
- 把“存储位置”“状态分层”“缓存策略”“一致性要求”分开
- 把“真实数据源”和“派生层”分开
- 如果草稿里直接跳到了具体技术，请回补前面的决策层
- 输出要适合长期维护

原始草稿如下：
【把草稿贴在这里】
```

## 高频补问

- `请先告诉我，我现在是在决定存储位置、状态分层，还是缓存策略。`
- `这个场景下哪些数据必须持久化，哪些只是运行时状态？`
- `缓存和真实数据源在这里应该如何区分？`
- `当前访问模式会如何影响选型？`
- `如果我优先追求简单实现，推荐会怎么变化？`
- `如果我优先追求一致性和长期维护，推荐会怎么变化？`
- `这个场景下最容易混淆的数据层概念是什么？`
