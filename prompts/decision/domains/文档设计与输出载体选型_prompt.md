# 文档设计与输出载体选型 Prompt

> **文档职责**：沉淀一套更贴近“文档设计 / 输出载体 / 写法组织”场景的决策 prompt。
> **适用场景**：需要决定文档用什么载体输出、如何组织结构、示例写到什么粒度、按什么叙述主线展开时。
> **目标读者**：本仓库维护者。
> **维护规范**：作为 `decision` 目录下的领域子 prompt 维护；当该主题下的概念、选项和判断框架稳定后，可继续拆分更细的子 prompt。

## 这个子 Prompt 解决什么问题

这类 prompt 主要解决下面几类常见困惑：

- 我只想到了 Markdown 和 HTML，但不知道还有没有别的输出方式
- 我不知道应该写成正式文档、可视化 artifact、还是分段讲解
- 我不知道示例应该给到最小可用、完整模板，还是场景化样例
- 我不知道应该按概念讲、按工具讲，还是按用户任务讲
- 我不确定这份文档是一次性参考、持续学习材料，还是 onboarding 材料

---

## 常见概念

这类问题里，最容易混在一起的概念包括：

- **输出载体**：最终交付给读者的形式是什么
- **文档结构**：内容按什么层次组织
- **叙述主线**：读者按什么理解路径阅读
- **示例粒度**：代码或配置要展示到什么详细程度
- **生命周期**：这份文档是一次性消费还是长期维护
- **读者角色**：面向新手、维护者、团队成员还是管理员

## 标准版

```text
请把这次对话当成一次“文档设计与输出载体选型”分析，不要急着替我拍板。

我当前要解决的问题是：
【把你的问题贴在这里】

我目前想到的方案是：
【写下你目前想到的方案；如果只有 Markdown / HTML 也照实写】

请按下面顺序帮我展开：

1. 先定义这个问题到底在决定什么
   - 是在决定输出载体
   - 还是在决定文档结构
   - 还是在决定写法组织
   - 如果我混在一起了，请拆开说明

2. 帮我补齐常见方案空间
   - 除了我已经想到的方案外，这个场景下还有哪些常见路径
   - 请至少列出 A / B / C
   - 如果存在“新手容易漏掉但很有价值”的方案，也请补上

3. 帮我澄清核心概念
   - 输出载体、文档结构、叙述主线、示例粒度、生命周期分别是什么意思
   - 哪些概念经常被混用
   - 它们分别影响什么

4. 比较这些方案
   - 分别从理解成本、维护成本、表达效果、可复用性、适用场景来比较
   - 明确每种方案更适合什么，不适合什么

5. 结合我的场景给出建议
   - 当前更推荐什么方向
   - 为什么
   - 这个建议依赖哪些前提
   - 如果目标从“临时输出”变成“长期维护”，推荐会怎么变化

6. 如果现在还不适合拍板
   - 请列出我最该继续确认的 3 到 5 个问题

输出要求：
- 先补齐方案空间，再进入推荐
- 如果我默认漏掉了行业常见方案，请明确指出
- 不要只说“哪个好”，要说“为什么这个场景更适合它”
- 如果我的问题其实跨了两个层级，请拆成两个决策问题
```

## 思路决策文档版

```text
请把下面这份关于“文档设计 / 输出载体 / 写法组织”的草稿，整理成一份“思路决策文档”。

不要把它写成最终拍板结论，而是写成一个可持续迭代的判断文档。

请按下面结构输出：
1. 这个问题在决定什么
2. 核心概念与边界
3. 常见方案空间
4. 比较维度
5. 当前草稿已提到的方案
6. 当前草稿遗漏但值得补充的方案
7. 当前待确认问题
8. 阶段性倾向
9. 后续建议继续追问的方向

要求：
- 保留不确定性
- 把“概念”与“结论”分开
- 把“方案空间”与“当前推荐”分开
- 如果草稿里只出现了熟悉方案，请补上容易遗漏的常见方案

原始草稿如下：
【把草稿贴在这里】
```

## 高频补问

- `我当前只想到了 Markdown 和 HTML，请帮我补齐这个主题下常见的输出载体。`
- `请先告诉我，我现在是在做“输出形式决策”还是“文档结构决策”。`
- `交互式 HTML artifact 这种方案属于什么类型，它适合什么场景？`
- `请把“输出载体”和“叙述主线”分开解释。`
- `请告诉我这份文档更像一次性参考、持续学习材料，还是 onboarding 材料。`
- `如果我要长期维护，这个选择会怎么变化？`
- `示例粒度应该如何分层，最小可用、完整模板、场景化示例分别适合什么情况？`

```text
## 8. 后续可继续扩展的方向
```

## 当前判断

- 这是对母 prompt 的领域化收窄
- 它适合成为你当前高频使用的第一批子 prompt
- 后续如果“输出载体”“文档生命周期”“示例粒度”持续增长，可以再从本文件拆出更细的 prompt
