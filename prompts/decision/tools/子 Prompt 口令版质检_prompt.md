# 子 Prompt 口令版质检 Prompt

> **文档职责**：为 `decision/domains/` 下的候选子 prompt 提供一句话级别的临时质检口令。
> **适用场景**：你正在和大模型对话，不想贴完整模板，只想先丢一句话让它做方向判断时。
> **目标读者**：本仓库维护者。
> **维护规范**：保持极短、可复制、低心智负担；不要把它扩写成快速版或完整版。

## 这份文件的定位

这是 `tools/` 里的最低成本入口。

它不负责完整质检，只负责一句话触发下面这类判断：

- 这个候选文件值不值得继续看
- 它更像独立子 prompt，还是更适合并到别处
- 它现在最大的问题是什么

默认作用域：

- 默认检查对象是 `decision/domains/` 下的候选领域子 prompt
- 不默认检查 `README.md`
- 不默认检查 `decision/core/`
- 不默认检查 `decision/tools/`

---

## 口令 1：最常用版

```text
请把下面这份文档当成 `decision/domains/` 的候选子 prompt，快速判断它值不值得收录。

只回答：
1. 值不值得继续做完整质检
2. 最大问题是什么
3. 更适合独立收录 / 修改后再收 / 并入现有文件 / 暂缓

文档如下：
【把文档贴在这里】
```

---

## 口令 2：边界判断版

```text
请快速判断下面这份候选文档，作为 `decision/domains/` 子 prompt 时，边界是否清楚。

只回答：
1. 主决策对象是否明确
2. 最可能和哪个现有文件重叠
3. 是否值得继续做完整质检

文档如下：
【把文档贴在这里】
```

---

## 口令 3：收录判断版

```text
请快速判断下面这份文档能不能收进 `decision/domains/`。

只给结论：
- 可以继续
- 修改后继续
- 不建议收录

再补一句理由。

文档如下：
【把文档贴在这里】
```

## 何时用它

优先用口令版的场景：

- 你正在对话里临时试探一个候选文件
- 你懒得贴完整快速版
- 你只想先拿到一个方向结论

如果出现下面情况，就升级：

- 你需要 5 项结构化判断
  用 [子 Prompt 快速质检_prompt.md](./%E5%AD%90%20Prompt%20%E5%BF%AB%E9%80%9F%E8%B4%A8%E6%A3%80_prompt.md)
- 你准备正式收录
  用 [子 Prompt 质检_prompt.md](./%E5%AD%90%20Prompt%20%E8%B4%A8%E6%A3%80_prompt.md)

---

## 当前判断

- 这份文件适合做 `tools/` 里的最低成本质检入口
- 它和快速版、完整版的关系是“先口令，再快速，再完整”
- 如果你后续发现某一句口令用得最多，可以把那一句再提到最前面
