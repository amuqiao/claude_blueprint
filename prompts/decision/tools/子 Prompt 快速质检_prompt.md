# 子 Prompt 快速质检 Prompt

> **文档职责**：为 `decision/domains/` 下的候选子 prompt 提供 30 秒级快速收录判断。
> **适用场景**：你不想先做完整质检，只想快速判断“值不值得继续看”“能不能先收”“更适合并到哪里”时。
> **目标读者**：本仓库维护者。
> **维护规范**：保持极简，只做快速分流；不要把完整版质检内容重新复制进来。

## 这份文件的定位

这不是完整版质检。

它的作用是先帮你做一个低成本判断：

- 这个候选文件值不值得继续做完整质检
- 它大致适合收录、修改后收录，还是暂不收录
- 如果不适合独立存在，更可能并到哪里

默认作用域：

- 默认检查对象是 `decision/domains/` 下的候选领域子 prompt
- 不默认检查 `README.md`
- 不默认检查 `decision/core/`
- 不默认检查 `decision/tools/`

## 直接可用版 Prompt

```text
请帮我快速检查下面这份文档，判断它是否适合收进 `decision/domains/`，作为一个独立子 prompt。

不要做完整质检，也不要大段润色，只做快速判断。

请只回答这 5 件事：
1. 它的主决策对象是否明确
2. 它是否和现有子 prompt 明显重叠
3. 它是否具备基本稳定结构
4. 它是否符合 `decision` 风格（先补盲、再比较、再推荐）
5. 最终建议是：
   - 可以进入完整质检
   - 修改后再进完整质检
   - 暂不建议收录

如果你认为不适合，请直接告诉我它更适合：
- 并入现有子 prompt
- 放进补问模板
- 做成 rule / checklist
- 继续作为草稿保留

输出要求：
- 每项都尽量短
- 优先指出最关键的 1 个问题
- 不要展开成完整审稿报告

候选文档如下：
【把文档贴在这里】
```

## 超短版 Prompt

```text
请快速判断下面这份文档是否值得收进 `decision/domains/`。

只回答：
1. 是否值得继续做完整质检
2. 最大问题是什么
3. 更适合收录 / 修改后收录 / 并入别处 / 暂缓

候选文档如下：
【把文档贴在这里】
```

## 何时用它

优先用这份快速质检的场景：

- 你刚起草完一个文件，只想先看方向对不对
- 你连续写了多个候选文件，想先筛掉明显不合适的
- 你不想每次都跑完整版质检

如果出现下面情况，就切到完整版：

- 你准备正式收进 `domains/`
- 你已经感觉边界重叠
- 你需要更具体的结构问题和修改建议

完整版入口：

- [子 Prompt 质检_prompt.md](./%E5%AD%90%20Prompt%20%E8%B4%A8%E6%A3%80_prompt.md)
