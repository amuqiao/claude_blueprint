# FastAPI 接入安全规则

FastAPI 服务必须先判断接口暴露面和调用主体，再选择凭证、授权、限流和浏览器接入策略。CORS 不是鉴权，HTTPS 不是授权。

## 暴露面判断

每个服务必须明确：

- 是否对公网开放。
- 调用方是内部服务、第三方系统、浏览器用户还是平台后端。
- 服务是否有用户系统、租户、资源归属或角色权限。
- 是否存在文件产物、长连接、Webhook 或回调入口。

内部临时服务也应有最小访问边界，不应默认裸奔到可被外部访问的网络。

## 凭证选择

| 场景 | 推荐凭证 |
| --- | --- |
| 内部轻量服务、单调用方 | `X-API-Key` |
| 多调用方但无用户系统 | 调用方级 API Key |
| 有用户、租户、资源归属 | JWT / Session + 授权策略 |
| 平台网关已鉴权 | 校验可信身份透传和网关边界 |

API Key 应支持轮换，不应写死在代码、镜像或前端产物中。生产密钥必须通过 Secret 或平台配置注入。

## 授权与限流

认证通过只代表“是谁”，不代表“能做什么”。存在资源归属、租户、配额、角色或付费边界时，必须增加授权判断。

对外服务、模型调用服务和高成本接口应考虑限流或配额。限流失败应返回明确错误，不暴露内部队列、模型供应商或密钥细节。

## 浏览器边界

浏览器接入需要同时考虑：

- CORS 只控制浏览器跨域读取，不是鉴权。
- Cookie 鉴权需要考虑 CSRF。
- 前端不得持有生产级服务密钥。
- 长连接、SSE、WebSocket 要明确断线、重连、鉴权和错误暴露策略。

## 失败可观测

鉴权和授权失败必须可排查，但不能泄露敏感信息：

- 401 表示未认证或凭证无效。
- 403 表示认证成功但无权限。
- 日志记录调用方摘要、request_id、失败原因分类。
- 不记录完整密钥、token、请求体中的敏感字段。
