# 执行清单规则

- 执行清单应放在对应项目或版本的文档链路目录中维护，不放在 `.agents/references/`。
- 文件名应保持稳定、可预测，并体现阶段语义。
- 执行清单只保留可直接开工的事项，不重复上游长文档。
- 每项任务应映射到明确动作，并附推荐顺序或最小完成标准。
- 执行清单应明确引用上游需求、范围、开发计划或实现文档。
- 同一阶段内默认原地更新当前清单；只有阶段目标明显变化时才新建下一份清单。
- 新清单应有新的阶段语义，不复用旧文件名硬改目标。
- 旧清单默认不删除；应标记 `active`、`completed` 或 `archived` 中的适当状态。
- `active` 表示仍在执行，`completed` 表示阶段完成但仍有回溯价值，`archived` 表示明确停止维护、仅保留存档价值。
- 建议在清单开头补充状态、适用阶段、上游文档和维护方式。
- 如果实现边界发生变化，应优先回写上游正式文档，再同步更新清单。
