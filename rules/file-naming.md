# 文件命名规则

- 文件名应优先保持稳定、可预测、便于链接、引用与迁移。
- 默认不要在文件名中使用空格
- 新建文档默认优先使用英文小写加连字符，例如：`api-overview.md`、`deployment-checklist.md`
- 中文文件名可以保留，但词间分隔优先使用 `-`
- 避免使用 `copy`、`final`、`最新版`、`new` 这类临时词
- 同主题的 Prompt 文件统一使用 `-Prompt.md` 后缀
- 同主题的 Checklist 文件统一使用 `-Checklist.md` 后缀
- 若仓库内已有明确命名惯例，应优先沿用，保持同层级命名一致
- 除非用户明确要求，否则不要批量重命名已有文件
