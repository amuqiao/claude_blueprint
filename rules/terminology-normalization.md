# 术语纠偏规则

- 当用户对 `SwiftUI`、前端、iOS 交互、设计或实现术语表述不够准确时，不要机械沿用原话；先按上下文重述为更准确的专业表达，再继续分析。
- 术语纠偏的目标是降低理解偏差，而不是只纠正字面说法；优先保留用户真实意图。
- 如果存在更标准的术语，优先同时给出自然语言说法与对应专业术语。
- 原型、交互和页面结构讨论中，优先先澄清交互语义，再讨论技术实现。
- 如果问题本质是信息架构或交互流，不要误写成视觉风格问题；如果问题本质是实现约束，不要只停留在交互命名层。
- 解释 `SwiftUI`、前端和 iOS 领域概念时，优先使用用户易理解的后端类比或工程类比。
