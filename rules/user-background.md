# 用户背景与理解上下文

## 1. 基本背景

- 用户名字是王桥，默认直接称呼“桥”
- 默认时区是 `Asia/Shanghai`

## 2. 技术背景判断

- 用户有 10 年 Python 后端经验，擅长 FastAPI 架构设计
- 不需要解释后端基础概念，如 HTTP、依赖注入、接口设计、数据库建模
- 用户对 AI 工程链路有一定了解，但不默认等同于对具体模型细节、训练机制或数学推导层已经精通
- 用户前端方向相对薄弱，解释 `React` / `TypeScript` 等概念时，优先用后端类比帮助建立认知

## 3. 理解偏好

- 回答 AI 相关问题时，默认先对齐所处层级：概念理解、工程实现、架构设计、训练细节、数学原理
- 遇到偏模型原理、训练细节或数学推导的问题时，优先给可理解版本，再按需补充更深入说明
- 解释 `React` / `TypeScript` 等前端概念时，优先用后端类比帮助建立理解

## 4. 默认技术栈

- 后端默认技术栈：`FastAPI` / `SQLAlchemy` / `Celery` / `Redis`
- 前端默认技术栈：`React` / `TypeScript` / `Zustand`
- 基础设施默认技术栈：`Docker` / `Nginx` / `PostgreSQL`
- 当用户未额外指定技术方案时，可优先基于上述栈给出实现、示例与架构建议
